# 01 项目背景与需求规格说明书

## 1. 项目背景

高校信息化经过 10+ 年建设已积累了教务、学工、财务、科研、人事、OA、一卡通、图书、网络等数十套业务系统，形成典型的"数据烟囱"：

- **数据孤岛**：各业务系统独立建库，跨系统数据难以打通；
- **口径不一**：同一指标（如"在校生人数"）在不同系统中口径差异 5%~15%；
- **共享低效**：部门间数据申请走邮件/线下审批，流程 7~15 个工作日；
- **决策滞后**：校领导驾驶舱依赖人工报表，T+3 才能看到上周数据；
- **合规压力**：《数据安全法》《个人信息保护法》《教育移动互联网应用程序备案管理办法》落地，等保三级测评在即。

本项目建设高校数据中台，作为全校数据的**统一采集、存储、治理、共享、服务、可视化、安全合规**底座。

## 2. 建设目标

| 层级 | 目标 |
| --- | --- |
| **战略目标** | 打造"一数一源、一源多用"的校级数据底座，成为智慧校园的数据大脑 |
| **业务目标** | 跨部门数据共享审批时长由周级压缩到小时级；校领导驾驶舱实现 T+0/T+1 |
| **技术目标** | 支持 50 个业务系统接入、10TB 总存储、99.9% 可用性、100 并发 API、P95 50ms |
| **合规目标** | 通过等保三级测评，满足《数据安全法》《个保法》《教育行业信息系统安全等级保护定级指南》 |
| **组织目标** | 沉淀数据资产目录、指标中心、标签中心、主数据，形成可复用数据资产 |

## 3. 范围界定（Scope）

### 3.1 In-Scope（本项目建设内容）

- 数据采集、数据仓库、数据治理、数据推送、数据服务、数据集市、数据安全、数据可视化 **八大功能模块**；
- 主数据管理、数据资产目录、指标中心、标签中心 **四大中心**；
- 统一门户 + 基于 CAS/LDAP 的单点登录；
- 完整流程引擎（审批、变更、数据订阅）；
- 内置大屏设计器（可视化拖拽）；
- 完整离线部署包（含国产 OS 适配）。

### 3.2 Out-of-Scope（本期不建设）

- 双活 / 同城灾备 / 异地灾备（仅本地备份恢复）；
- 多校区多租户（单校单实例）；
- 业务系统本身的改造（仅作为数据源/消费方对接）；
- 数据科学平台（Notebook、MLOps）—— 规划二期；
- 统一身份认证系统的建设（直接对接学校已有 CAS/LDAP）。

## 5. 用户与组织模型

- **租户**：单租户（全校）。
- **组织机构**：从 CAS/LDAP 同步，支持"学校—学院/部门—科室—个人"四级；
- **角色预置**：
  - 系统管理员（sysadmin）
  - 安全管理员（secadmin）
  - 审计管理员（auditadmin）—— 三权分立
  - 数据管理员（dataadmin）
  - 数据开发者（developer）
  - 部门数据管理员（dept_dm）
  - 业务用户（biz_user）
  - 师生访客（public_user）

> **等保三级要求**三权分立，系统/安全/审计管理员不得兼任，详见 `06-安全与合规/01-等保三级合规矩阵.md`。

## 7. 非功能性需求

### 7.1 性能
- API：P95 ≤ 50 ms，P99 ≤ 200 ms，100 并发不降级；
- 大屏首屏：≤ 3 s，组件刷新 ≤ 1 s；
- 实时采集：延迟 ≤ 5 s（源库 commit → ODS 可见）；
- ETL 批处理：10 GB/h 吞吐；
- 指标查询：预聚合下 P95 ≤ 500 ms。

### 7.2 可用性
- 整体 99.9%（年 ≤ 8.76 h 宕机）；
- 无状态服务：至少 2 副本 + 负载均衡；
- 有状态组件（PG/CK/Kafka/Redis/Elasticsearch）：主备、主从或集群部署 + 自动故障检测；
- 平台级恢复目标：RTO ≤ 2 h，RPO ≤ 15 min；其中 PostgreSQL 核心库通过 WAL 连续归档实现更细粒度恢复。

### 7.3 可扩展性
- 水平扩展：微服务 + Docker 容器部署（Docker Compose 编排）；
- 数据扩展：PG 声明式分区 + ClickHouse 分片；
- 接入扩展：采集插件化架构（SPI）。

### 7.4 安全性
- 等保三级全量合规（详见 `06-安全与合规/01-等保三级合规矩阵.md`）；
- 国密算法：传输 SM2 国密 TLS、存储 SM4、签名 SM2 + SM3；
- 三权分立、最小权限、审计不可篡改。

### 7.5 可维护性
- 集中日志（Loki/ELK）+ 指标（Prometheus + Grafana）+ 链路（SkyWalking）；
- 配置中心统一（Nacos），禁止硬编码；
- 健康检查、优雅停机、灰度发布。

### 7.6 兼容性
- OS：麒麟 V10 SP3、统信 UOS V20（x86_64 主，ARM 兼容）；
- CPU：海光、飞腾、鲲鹏；
- 浏览器：Chrome/Edge 最近 2 个主版本、360 安全浏览器；
- JDK：BiSheng / OpenJDK 17；
- 中间件：全部选择支持国产 OS 的开源版本。

### 7.7 本地化
- 文档、界面、报错信息全中文；
- 时间：Asia/Shanghai；
- 编码：UTF-8；
- 字符集：支持中文（简/繁）、少数民族文字、生僻字（GB18030/UTF-8）。

## 8. 约束与前提

| 类别 | 约束 |
| --- | --- |
| 技术栈 | Java + Spring Cloud Alibaba + Nacos + PostgreSQL（指定） |
| 网络 | 校内部署，数据域与管理域隔离，对外通过反代白名单 |
| 身份 | 对接学校 CAS（首选）+ LDAP，无需自建用户体系 |
| 许可证 | 默认优先选用 Apache 2.0 / MIT / BSD 等商业友好协议；如确需引入 AGPL / Elastic License / SSPL 类组件，必须经学校法务与项目架构组联合评审并形成白名单说明 |
| 部署 | 生产环境支持离线部署：学校服务器侧所有依赖镜像、包、字体、字典本地化；开发测试环境允许联网 |
| 版本 | 所有中间件采用近一年 LTS 稳定版，锁定版本号 |

## 9. 验收标准（摘要）

1. 八大模块全部通过功能测试，**核心模块单元测试行覆盖率 ≥ 70%、关键主流程集成测试 100% 覆盖**（与 `08-开发规范/01-编码规范.md#六` 与 `10-项目管理/01-WBS与开发排期.md#1` 口径一致）；
2. 性能压测达标：100 并发 API P95 ≤ 50ms，10GB/h ETL；
3. 等保三级测评通过（第三方测评机构出具报告）；
4. 代码扫描无高危漏洞（SonarQube + 依赖扫描 + 国密合规扫描）；
5. 文档齐备：本 docs 目录全量交付 + 运维手册 + 用户手册；
6. 培训完成：管理员 2 期、开发者 1 期、业务用户 2 期。
