# 02 核心假设与选型决策

本文件将所有"不在用户明确指定范围内、但必须做出决策"的项逐一列出，每项给出"选型结论 + 备选方案 + 决策理由 + 变更影响"。后续如需调整，可独立迭代。

---

## A. 数据存储栈

### A.1 分析型存储
- **结论**：PostgreSQL 15（ODS / DWD / DIM 明细层）+ ClickHouse 24.x（DWS / ADS 分析层）。
- **备选**：Greenplum；Apache Doris；单纯 PG + Citus。
- **理由**：
  1. 总量 10TB、日增 10GB 场景下，PG 声明式分区 + 物化视图 可承载 ODS/DWD；ClickHouse 专治宽表聚合，大屏/BI 查询 10x~100x 加速；
  2. 两者均有完善的国产 OS 兼容与离线镜像（openEuler/麒麟均有官方包）；
  3. Greenplum 资源占用高；Doris FE/BE 模型复杂，运维成本高；单纯 Citus 在写密集 CDC 下容易成为瓶颈；
  4. 冷数据可通过 `clickhouse-backup` + MinIO 归档。
- **变更影响**：大部分 ETL、指标 SQL 只需改数据源，血缘图与分层规范不变。

### A.2 对象存储
- **结论**：MinIO（最新稳定版，S3 协议，分布式纠删码部署）。
- **备选**：Ceph RGW；FastDFS；学校现有存储。
- **理由**：MinIO 单二进制部署、零外部依赖、支持 KMS 国密扩展、S3 协议与主流 SDK 兼容，天然适合单校单实例离线场景。
- **容量规划**：一期按 4 节点、每节点 4 盘规划，有效容量按 16~24 TB 预估；具体纠删码条带随磁盘数统一编排，避免文档口径与实际部署不一致。

### A.3 缓存
- **结论**：Redis 7.2（主从 + Sentinel，3 节点）。
- **用途**：API 响应缓存、Token、限流计数、热点指标、分布式锁。

### A.4 检索
- **结论**：Elasticsearch 8.x。
- **用途**：数据资产目录全文检索、日志审计检索、非结构化元数据搜索。
- **离线方案**：需额外下载 icu/analysis-ik 中文分词包。

### A.5 图数据库（可选，二期可下放）
- **结论**：一期不强制，预留 Nebula Graph v3 接入口，用于数据血缘可视化下钻。
- **一期替代**：血缘以 PG 表 + D3.js 前端渲染，足够覆盖字段级血缘。

## B. 数据流转栈

### B.1 消息队列
- **结论**：Apache Kafka 3.7（KRaft 模式，去 ZooKeeper）。
- **备选**：RocketMQ 5；Pulsar。
- **理由**：
  1. Flink CDC 对 Kafka 兼容最佳，社区生态丰富；
  2. KRaft 模式简化了部署，3 节点即可；
  3. RocketMQ 在大数据场景生态较弱；Pulsar 运维复杂度偏高。

### B.2 实时计算
- **结论**：Apache Flink 1.18（Standalone，基于 Checkpoint/Savepoint 与调度重拉起的一期容灾方案，容器化部署）。
- **用途**：CDC 流、实时聚合、实时质量监控。

### B.3 CDC 采集
- **结论**：Flink CDC 3.x（内含 Debezium 引擎），全量 + 增量一体。
- **备选**：Canal（仅 MySQL）、Debezium 独立部署、DataX（无 CDC）。

### B.4 批量采集/推送
- **结论**：DataX（阿里开源）+ 自研插件。
- **理由**：DataX 的 Reader/Writer 插件化、支持全量、内存占用低；对达梦、金仓有官方或社区插件。
- **补充**：SeaTunnel 作为二期候选，一期不强制引入，避免技术栈膨胀。

### B.5 工作流/调度
- **结论**：Apache DolphinScheduler 3.2（DS）。
- **备选**：Apache Airflow；XXL-Job。
- **理由**：DS 为国产，中文友好，可视化 DAG、支持 SQL/Shell/Python/HTTP/DataX/Flink 任务，完全满足"定时+实时+依赖+补数+告警"。
- **审批流引擎**：Flowable 7（BPMN 2.0），独立于 DS。

## D. 密码学与安全

- **算法**：全量国密 —— SM2（非对称 / 签名）、SM3（哈希）、SM4（对称 / ECB 严禁、CBC/GCM 推荐）。
- **TLS**：内部服务 mTLS 国密 SSL（`tlcp`），基于 BabaSSL / TongSuo；若运维难度高，允许内部使用 TLS 1.3 + RSA，网关层 SM2 国密 SSL 对外。
- **KMS**：自研基于 PostgreSQL + HSM/软 HSM 的 KMS 服务；生产环境建议对接学校硬件加密机（如三未信安、江南天安）。
- **密钥分层**：主密钥 MK（HSM 托管）→ 数据加密密钥 DEK（按业务域派生）→ 字段密钥 FEK（按分级派生）。

## E. 审批流程设计（基于数据分级动态路由）

### E.1 数据分级
| 级别 | 名称 | 示例 |
| --- | --- | --- |
| L1 | 公开 | 学院数量、公共课程目录 |
| L2 | 内部 | 部门统计、非敏感聚合 |
| L3 | 敏感 | 师生姓名+工号+手机、成绩、工资 |
| L4 | 机密 | 身份证号、银行卡号、病历、密码 Hash、人事档案 |

### E.2 审批角色
- 申请人（业务用户）
- 资源属主部门管理员（dept_dm）
- 数据管理员（dataadmin）
- 安全管理员（secadmin）
- 审计管理员（auditadmin）—— 事后审计，不参与流程

### E.3 动态路由规则

| 分级 | 审批层级 | 流程 |
| --- | --- | --- |
| L1 公开 | 0 级 | 自动通过 + 审计记录 |
| L2 内部 | 1 级 | 资源属主部门管理员 |
| L3 敏感 | 2 级 | 资源属主部门管理员 → 数据管理员 |
| L4 机密 | 3 级 | 资源属主部门管理员 → 数据管理员 → 安全管理员 |

### E.4 特殊规则
- 跨部门访问任何级别自动升一级；
- 申请批量字段中存在多级别，按最高级别定级；
- 审批时限：1 级 24h、2 级 48h、3 级 72h，超时自动催办并通知上级；
- 一键撤销：授权生命周期 30/90/180/365 天可选，到期自动回收；
- 所有审批节点留痕，节点信息全部进入审计表，不可删除。

详细 BPMN 定义见 `06-安全与合规/04-审批流程设计.md`。

## F. 目标运行环境

### F.1 操作系统
- **主**：麒麟 V10 SP3（x86_64）
- **次**：统信 UOS V20（x86_64）
- **预留**：麒麟 V10 SP3（ARM/鲲鹏）—— 二期交叉编译

### F.2 硬件基线（建议，非强制；与 `01-总体架构/04-部署架构与网络设计.md#3 / #8` 保持一致）
| 角色 | 数量 | 规格 |
| --- | --- | --- |
| 应用服务器（Docker，运行 20 个微服务，多副本） | 5 | 16C/64G/500G SSD |
| PostgreSQL 业务实例（1 主 + 1 同步备 + 1 异步备/读） | 3 | 16C/64G/2T NVMe + 4T SATA |
| PostgreSQL 数仓实例（1 主 + 1 备） | 2 | 16C/64G/4T NVMe |
| ClickHouse 节点（2 分片 × 2 副本，使用内置 keeper 模式 3 台仲裁） | 4 + 3 | CK：16C/128G/4T NVMe；keeper：4C/8G/200G |
| Kafka（KRaft 模式 3 节点） | 3 | 8C/32G/2T SSD |
| Flink（JobManager × 2 + TaskManager × 3） | 5 | 16C/64G/500G |
| MinIO（4+0 EC，4 节点起步） | 4 | 8C/32G/6T HDD × 4（按实际磁盘数做纠删码编排） |
| Redis（Sentinel 3 节点） | 3 | 8C/16G |
| Elasticsearch（3 节点集群） | 3 | 8C/32G/2T SSD |
| 管理节点（DS / Harbor / 监控 / 堡垒机 / Nacos） | 3 | 8C/32G/500G |
| 合计 | ≈ 38 台 | ≈ 480C / 1.8TB RAM / 60TB 存储 |

> 容量规划口径与 `01-总体架构/04-部署架构与网络设计.md#3 节点拓扑、#8 容量规划` 完全对齐。学校实际部署可按"应用合并、PG 共享实例"等方式精简至 14~18 台，但 PG 业务/数仓实例物理分离与 ClickHouse 分片副本结构不可调整，否则影响合规与性能。

> 一期采用 Docker 容器化部署，可按环境规模拆分为若干物理机或虚拟机；后续如学校运维能力提升，再评估向更复杂编排平台演进。

### F.3 网络
- 网络分区统一为：DMZ（VLAN5）/ 管理域（VLAN10）/ 应用域（VLAN20）/ 服务域（VLAN30）/ 数据域（VLAN40）/ 外联域（VLAN50），详见 `01-总体架构/04-部署架构与网络设计.md`。
- 对外仅暴露：SSO 回调、门户 443、API 网关 443、大屏 443；均经反代 + WAF。
