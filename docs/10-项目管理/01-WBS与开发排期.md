# 01 WBS 与开发排期

## 1. 总体参数

| 项 | 值 |
| --- | --- |
| 项目总周期 | 8 个月（约 34 周） |
| 里程碑 | M0 启动 / M1 需求冻结 / M2 架构评审 / M3 基础底座 / M4 核心模块 / M5 全量功能 / M6 UAT / M7 试运行 / M8 正式上线 |
| 团队 | 10 人：架构师 1、后端 4、数仓 2、前端 2、测试 1 |
| 质量门 | 单元 ≥ 70%、集成覆盖核心主流程、Sonar 无 Blocker/Critical、通过第三方等保三级预评 |
| 交付 | 源码 + docs + 离线安装介质 + 培训 |

## 8. 验收标准

1. 功能矩阵 100% 覆盖；
2. 性能指标达标（API P95≤50ms, ETL≥10GB/h, 100 并发大屏流畅）；
3. 等保三级测评通过；
4. Sonar 无高危；OWASP Top 10 无中危以上漏洞；
5. 文档齐备；
6. 培训完成；
7. 7 天试运行无 P0/P1 事件。

## 9. 变更管理

- 需求变更单 → 影响分析 → 评审 → 计划调整；
- 冻结期（上线前 2 周）禁止新增需求，仅允许 Bug 修复。

## 10. 版本与迭代规划

- **V1.0 正式版**：本 8 个月交付；
- **V1.1**（+3 个月）：标签算法化、Nebula 血缘图、SeaTunnel、BI 自助升级；
- **V1.2**（+6 个月）：数据科学平台接入、模型服务、移动端完善；
- **长线**：多校协作、跨校数据交换、AIGC 增强问答。
