# 高校数据中台系统 研发落地设计文档

> 版本：V1.0 ｜ 编制日期：2026-04 ｜ 文档性质：研发落地级设计（开发即可据此启动）
>
> 目标：构建一套**符合高校业务场景、通过等保三级测评、可单校单实例私有化离线部署**的数据中台系统。
>
> 技术主栈：Java 17 + Spring Cloud Alibaba（Nacos/Sentinel/Seata）+ PostgreSQL + ClickHouse + Kafka + Flink + Redis + Elasticsearch + MinIO + Vue 3。

## 二、关键业务目标与技术指标

| 维度 | 指标 | 备注 |
| --- | --- | --- |
| 数据源接入 | 1~50 个业务系统 | 含关系型/NoSQL/文件/API/MQ |
| 日增量 | ≤ 10 GB / 日 | 结构化为主 |
| 总存储 | ≤ 10 TB | 结构化 6T + 非结构化 4T 规划 |
| 实时采集速率 | ≤ 10 000 条/分钟 | ≈ 167 QPS |
| ETL 吞吐 | ≥ 10 GB/小时 | 批处理峰值 |
| API 服务 | 100 并发，P95 ≤ 50 ms | 读场景为主 |
| 可用性 | 99.9% | 年宕机 ≤ 8.76 h |
| 大屏/门户 | 100 同时在线 | 含可视化 |
| 合规等级 | 等保三级 | 含国密算法、审计、分级分类 |
| 部署形态 | 单校单实例 + 完全离线 | 麒麟 V10 / 统信 UOS |

## 三、阅读顺序建议

- **架构师 / 技术经理**：按 `00 → 01 → 02 → 06 → 09` 的顺序通读，建立全景认知。
- **模块开发负责人**：`00-02-核心假设与选型决策.md` → `01-03-微服务拆分与服务清单.md` → 自身模块详设 → `04 数据库详设` → `05 接口清单`。
- **DBA / 数仓**：`02 数据架构` → `04 核心表结构清单` → `07 任务编排`。
- **安全工程师**：`06 安全与合规` → `03-07-数据安全模块.md`。
- **PMO**：`00 项目概述` → `10-项目管理/01-WBS与开发排期.md` → `10-项目管理/02-开发任务计划.md`。
- **研发组长 / 一线工程师**：`10-项目管理/02-开发任务计划.md`（含 P0 开发环境搭建 + P1~P8 每步任务与完成效果）。
