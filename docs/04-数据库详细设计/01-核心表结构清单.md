# 01 核心表结构清单（DDL 级）

> 约定：全部 PostgreSQL 15，UTF-8。所有表默认带 `id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY`、`create_time/update_time/create_by/update_by/is_deleted/version/tenant_id` 通用字段（通过 `bdc_common_base` 模板脚本注入）。本清单列出业务核心字段，省略部分通用字段以保持可读。
>
> **物理库口径**（与 `02-数据架构/02-数据库分库与表结构规范.md#1.0`、`09-部署运维/01-离线部署方案.md#4.6` 完全一致）：
> - 一期共 4 个 PG 库 + 1 个 CK 库：`bdc_app`（16 个业务 Schema）、`bdc_flowable`、`bdc_audit`、`bdc_dw`、`bdc_dw_olap`(CK)；
> - 文中出现的 `bdc_auth`、`bdc_uac`、`bdc_visual`、`bdc_collect` 等写法均表示 **`bdc_app` 库下的逻辑模块 Schema**，例如 `bdc_uac.t_user` 等价于 `bdc_app.uac.t_user`、`bdc_visual.t_page` 等价于 `bdc_app.visual.t_page`；
> - **仅** `bdc_flowable`、`bdc_audit`、`bdc_dw`、`bdc_dw_olap` 是真正的独立库；
> - 应用代码使用 MyBatis-Plus 时，统一通过 `bdc-common-db` 注入 `search_path`，业务无需关心物理库与 Schema 的拼接。

## 目录
- A. 平台底座：bdc_auth / bdc_uac / bdc_notify / bdc_audit / bdc_config
- B. 元数据与四大中心：bdc_metadata / bdc_mdm / bdc_indicator / bdc_tag
- C. 八大模块：bdc_collect / bdc_warehouse / bdc_govern / bdc_push / bdc_dataservice / bdc_market / bdc_visual / bdc_security
- D. 审批：bdc_flowable（Flowable 官方）
- E. 数据仓库：bdc_dw（ODS/DWD/DIM 部分样例）

### A.1 `bdc_auth`

```sql
-- 用户登录快照（与 bdc_uac.t_user 对齐；本表仅做登录/令牌管理）
CREATE TABLE bdc_auth.t_login_log (
    id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL,
    login_name VARCHAR(64) NOT NULL,
    login_type SMALLINT NOT NULL,            -- 1=CAS,2=LDAP,3=密码,4=API
    login_ip VARCHAR(64),
    user_agent VARCHAR(512),
    login_time TIMESTAMPTZ NOT NULL DEFAULT now(),
    status SMALLINT NOT NULL,                -- 1=成功 0=失败
    fail_reason VARCHAR(256)
);
CREATE INDEX idx_login_log_user ON bdc_auth.t_login_log(user_id, login_time DESC);

CREATE TABLE bdc_auth.t_token (
    id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL,
    token_hash VARCHAR(128) NOT NULL,        -- SM3 Hash
    refresh_token_hash VARCHAR(128),
    expire_time TIMESTAMPTZ NOT NULL,
    ip VARCHAR(64),
    ua VARCHAR(512),
    status SMALLINT NOT NULL DEFAULT 1,
    create_time TIMESTAMPTZ DEFAULT now(),
    UNIQUE (token_hash)
);
```

CREATE TABLE dws_stu_profile_w ON CLUSTER 'bdc_ck' (

    org_code LowCardinality(String),
    grade LowCardinality(String),
    major_code LowCardinality(String),
    gpa Nullable(Float32),
