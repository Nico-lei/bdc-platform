## 2. 网络分区与安全域

| 域       | VLAN    | 用途                     | 进出规则                |
| ------- | ------- | ---------------------- | ------------------- |
| 接入域 DMZ | VLAN 5  | 反代 + WAF               | 仅 80/443 from 校园网   |
| 服务域     | VLAN 30 | 网关 + 前端                | 只接受 DMZ 进 + 管理域 SSH |
| 应用域     | VLAN 20 | 业务微服务 + 中间件客户端         | 只接受服务域 + 允许访问数据域    |
| 数据域     | VLAN 40 | 所有数据库/存储/Kafka/Flink   | 只允许应用域访问；无出站到互联网    |
| 管理域     | VLAN 10 | CI/CD、监控、日志、堡垒机        | 仅管理员通过 VPN/堡垒机进入    |
| 外联域     | VLAN 50 | 对接学校 CAS/LDAP、业务系统只读账号 | 仅白名单 IP & 端口        |

防火墙策略采用**默认拒绝 + 按需白名单**，所有跨 VLAN 访问必须在 IPS/IDS 日志中留痕（等保三级网络安全要求）。

## 3. Docker 部署拓扑

| 节点              | 数量 | 规格              | 角色                                  |
| --------------- | -- | --------------- | ----------------------------------- |
| 应用节点            | 5  | 16C/64G/500G    | Docker 容器运行 20 个业务服务                |
| 中间件节点           | 3  | 16C/64G/1T SSD  | Kafka(KRaft)/Redis Sentinel/Nacos/Sentinel/Flink |
| DB 节点（物理或独立 VM） | 12+ | 16C/64G/2T NVMe | PG 业务实例 ×3（主+同步备+异步备）；PG 数仓实例 ×2（主+备）；CK ×4（2 分片 × 2 副本）+ Keeper ×3；MinIO ×4 |
| 管理节点 | 3 | 8C/32G/500G | Harbor、DolphinScheduler Master/Worker、堡垒机、监控（Prometheus/Grafana/Loki/SkyWalking） |

> **有状态数据组件**一律**独立部署**，不纳入应用容器编排，便于运维与合规审计（等保三级对数据库审计要求高）。

## 4. 高可用策略

| 组件               | HA 方案                                     | 节点数 | RTO    | RPO      |
| ---------------- | ----------------------------------------- | --- | ------ | -------- |
| 业务服务             | Docker Compose 多副本 + Nginx/Gateway 切流     | 5（应用节点） | < 30s  | 0        |
| Gateway          | 2 副本 + Keepalived VIP                    | 2 | < 10s  | 0        |
| Nacos            | 集群模式 + 内嵌 Derby/外接 PG                     | 3 | < 30s  | 0        |
| PostgreSQL 业务实例   | 主备流复制 + Patroni + HAProxy + Keepalived    | 3（1 主 + 1 同步备 + 1 异步备） | < 60s  | < 5s     |
| PostgreSQL 数仓实例   | 主备流复制 + Patroni                          | 2（1 主 + 1 备） | < 90s  | < 30s    |
| ClickHouse       | 2 副本 × 2 分片（ReplicatedMergeTree + 内置 Keeper） | 4 CK + 3 Keeper | < 1min | 秒级       |
| Kafka            | KRaft（无 ZooKeeper）                       | 3 | < 30s  | 0        |
| MinIO            | 分布式部署 + 4+0 纠删码                          | 4 | < 30s  | 0        |
| Redis            | Sentinel                                  | 3 | < 30s  | < 1s     |
| Elasticsearch    | 集群（master/data/coordinator 同节点轻量化）        | 3 | < 60s  | < 30s    |
| Flink            | Checkpoint/Savepoint + DolphinScheduler 重拉起 | 2 JM + 3 TM | < 5min | Kafka 偏移 |
| DolphinScheduler | Master×2 + Worker×3 + PG 元库                | 5 | < 1min | < 1min   |
| Flowable         | 内嵌于 `bdc-approval` 服务（多副本无状态 + `bdc_flowable` PG 集群） | 应用节点 ×N | < 30s  | < 1s     |

## 7. 离线部署方案要点

1. **离线包准备**：
   - OS 基础包（yum 本地仓/dnf 私仓）
   - Docker / Containerd 离线 rpm 包
   - Harbor 私仓镜像（JDK、Nacos、PG、CK、Kafka、Flink、MinIO、Redis、Elasticsearch、DS、SkyWalking、Prometheus、Grafana、Loki、Nginx、前端构建产物、20 个业务服务镜像）
   - Maven 本地仓（Nexus）/ NPM 本地仓（Verdaccio）/ Python PyPI 镜像（Bandersnatch）
   - 字典/字体/GeoJSON 地图本地化资源
2. **安装介质结构**：

```
bdc-installer-v1.0.0/
├── os/                 # 基础 rpm
├── images/             # Harbor 镜像 tar
├── compose/            # Docker Compose 包
├── sql/                # 初始化 SQL
├── config/             # 各环境配置
├── scripts/            # 自动化安装脚本
└── docs/               # 本文档
```

1. **安装流程**（由运维按 `09-部署运维/01-离线部署方案.md` 执行）：
   - 环境检查 → OS 配置 → Docker/Compose 安装 → Harbor 导入 → 中间件安装 → 微服务部署 → 数据库初始化 → 监控接入 → 功能验证

## 9. 灾备策略（按需求不建双活）

- **本地冷备**：每日全备 PG + CK + MinIO → 独立备份存储（NAS 或 MinIO 异集群）；
- **WAL 增备**：PG WAL 1 分钟归档到备份存储；
- **MinIO**：开启版本化 + 桶级 LifeCycle（软删保留 30 天）；
- **Kafka**：保留 7 天 + Offsets 持久化；
- **备份保留策略**：日备 30 份、周备 12 份、月备 12 份、年备 3 份；
- **恢复演练**：季度一次，形成报告存档；

详见 `09-部署运维/03-备份恢复方案.md`。
