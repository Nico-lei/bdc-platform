# 01 任务编排与调度规范

## 1. 调度平台选型

- **DolphinScheduler 3.2**（DS）作为**核心批调度**；
- **Flink Application Mode**：长时运行的流式任务；
- **Spring @Scheduled + XXL-Job Executor（轻量）**：服务内部自身的轻量定时（如缓存刷新、状态巡检）；
- 统一门户通过 `bdc-scheduler-adapter` 封装 DS REST API。

## 2. 任务命名规范

```
<domain>_<action>_<object>[_<period>]

例：
ods_sync_jwxt_student       (ODS 采集 学生表)
dwd_build_teach_select_d    (DWD 选课宽表，日粒度)
dim_refresh_person          (维度刷新)
dws_agg_stu_profile_w       (学生画像周)
ads_kpi_leader_dashboard_d  (领导驾驶舱日 KPI)
dq_check_mdm_student        (主数据学生质量检查)
push_mysql_edu_stu_d        (推送到校外系统)
```

- `domain` 取 `ods/dwd/dim/dws/ads/dq/push/meta/backup/arch/mon` 之一；
- `action` 取 `sync/build/refresh/agg/kpi/check/push/scan/archive/monitor`；
- `object` 为主题/表关键词；
- `period` 取 `rt/i/h/d/w/m/y`（realtime/interval/hour/day/week/month/year）。

## 3. 工作流组织

### 3.1 工作流粒度

- 同一**主题域**、同一**粒度**、同一**调度节奏**的任务打包为一个工作流；
- 例：
  - `wf_stu_daily`：学生主题日任务；
  - `wf_teach_term`：教学学期任务；
  - `wf_fin_monthly`：财务月任务；
  - `wf_leader_hourly`：领导驾驶舱小时。

### 3.2 DAG 模式

- **典型顺序**：ODS 采集 → DQ → DWD → DIM → DWS → ADS → DQ → 通知；
- **并行**：同层级任务并行；
- **资源组**：按优先级划分（P0 保核心、P2 低优）。

### 3.3 依赖声明

DS 支持：
- **顺序依赖**：节点 A 完成后 B；
- **跨工作流**：B 等待 A 工作流完成；
- **数据到位**：文件/表事件触发（DS DataSource 检查器）；
- **成功 / 失败 / 任意** 分支。

## 4. 调度策略

### 4.1 调度时间窗口

| 类别 | 建议时段 |
| --- | --- |
| ODS 采集 | 实时 + 01:00 批 |
| DWD/DIM 构建 | 02:00~04:30 |
| DWS/ADS 聚合 | 04:30~06:00 |
| DQ 检查 | 每层构建完成后 |
| 领导驾驶舱刷新 | 06:00（T+1） + 09:00 增量 |
| 推送外部 | 07:00~09:00 |
| 备份 | 02:00 开始 |

> 具体时间由项目运行期根据资源占用统计调整。

### 4.2 SLA 等级

| 等级 | 说明 | 重试次数 | 告警 | 延迟阈值 |
| --- | --- | --- | --- | --- |
| P0 | 核心（主数据、领导驾驶舱） | 3 | 钉钉+短信+邮件 | 30 min |
| P1 | 重要（业务主题） | 3 | 钉钉+邮件 | 1 h |
| P2 | 一般 | 2 | 邮件 | 2 h |
| P3 | 自助探索 | 1 | 平台消息 | 当日处理 |

## 5. 任务类型与模板

### 5.1 SQL 任务模板（PG）
```sql
/* @job: dwd_build_teach_select_d @period: day */
INSERT INTO dwd_teach.dwd_course_select_d (...)
SELECT ...
FROM ods_jwxt.t_course_select o
LEFT JOIN dim.dim_person p ON ...
WHERE o._etl_ts >= date_trunc('day', now()) - interval '1 day'
ON CONFLICT (select_id) DO UPDATE SET ...;
```

### 5.2 SQL 任务模板（ClickHouse）
```sql
INSERT INTO dws_stu_profile_w
SELECT ... FROM dwd_teach.dwd_course_select_d WHERE ...
SETTINGS max_insert_block_size = 100000;
```

### 5.3 Python 任务
```python
# @job: dq_check_stu_idcard
import os, psycopg2
conn = psycopg2.connect(os.environ['PG_DW'])
with conn.cursor() as cur:
    cur.execute("SELECT count(*) FROM dim.dim_person WHERE is_current AND length(id_card_hash) != 64")
    bad = cur.fetchone()[0]
print({"bad_count": bad})
```

### 5.4 Shell 任务（尽量少用）
- 仅允许调用白名单二进制：`pgbackrest, mc (mc mirror), clickhouse-backup, pg_dump, pg_restore`；
- 禁用：`rm`, `curl 外网`, `wget 外网`。

### 5.5 DataX 任务
- DS "DataX 任务" 组件；
- Reader / Writer 插件化；
- 参数 JSON 中禁止明文密码：使用 `${jdbc.password}`（从 DS 环境变量注入，来自 KMS）。

### 5.6 Flink 任务
- DS "Flink 任务" 组件；
- 以 Application Mode 提交，`jobName = <wf>_<task>`；
- 状态后端：RocksDB + HDFS（或 MinIO S3 协议）；
- Checkpoint：Exactly-Once，10s 间隔。

## 6. 参数化与变量

### 6.1 全局变量（DS 级）
- `${BATCH_DATE}`：业务批次日期（默认 yesterday）
- `${BATCH_ID}`：批次 UUID
- `${TERM}`：当前学期代码
- `${ACADEMIC_YEAR}`：学年
- `${ENV}`：`prod/test/dev`
- `${DS_HOME}`：DS 根
- `${PG_DW}` / `${CK_DW}`：连接串（KMS 注入）

### 6.2 工作流变量
- 在工作流中声明，子任务继承。

### 6.3 运行参数
- 手动运行时可覆盖参数；
- 补数模式：传入 `from=2026-01-01 to=2026-03-31 parallel=4`。

## 7. 补数与重跑

### 7.1 补数
- 界面化：选择工作流、日期范围、并行度；
- 生成 N 个实例，依赖共享；
- 失败只补失败实例；

### 7.2 重跑
- 从失败节点起；
- 或从最顶层（强制）；
- 所有重跑自动记录 `trigger=manual`，并审计。

## 8. 监控与告警

### 8.1 DS 自带
- 工作流状态、任务状态、运行耗时、当前等待；
- 工作流失败即告警（邮件/钉钉）。

### 8.2 平台集成
- 通过 `bdc-scheduler-adapter` 将状态同步给 `bdc-govern` / `bdc-market`；
- Prometheus 采集 DS Master/Worker 指标；
- 失败率 > 10% 告警；
- P0 任务失败立即触发"值班响应"流程。

## 9. 补数/重跑审计

- 每次补数 / 重跑写 `bdc_govern.t_backfill_log`；
- 管理员审批后方可批量补数（防止误操作）。

## 10. 最佳实践

1. **幂等**：所有任务支持重复执行不产生重复数据；
2. **原子**：一个任务只做一件事；
3. **隔离**：敏感任务单独资源组，与普通任务分开；
4. **超时**：所有任务声明超时（默认 1h），超时告警；
5. **日志**：任务日志采集到 Loki，按 `wf/task/date` 归集；
6. **参数回显**：执行开始/结束时打印所有参数；
7. **失败快速**：质量阻断规则 → 失败即止，阻止脏数据下游；
8. **资源预算**：每任务声明 CPU/内存（DS 资源标签）；
9. **锁**：跨任务共享资源加分布式锁（Redis）避免并发；
10. **版本**：任务脚本随 Git 版本 Tag；发布时写入元数据。

## 12. 资源组与优先级

| 资源组 | 机器 | 优先级任务 |
| --- | --- | --- |
| core | worker-core-01~02 | P0（领导、主数据） |
| biz | worker-biz-01~03 | P1/P2（业务主题） |
| adhoc | worker-adhoc-01 | P3（用户自助、探索） |
| stream | （Flink 集群） | 实时任务 |

资源组通过 DS worker group 实现，任务创建时指定。
