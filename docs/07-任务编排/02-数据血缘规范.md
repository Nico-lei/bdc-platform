# 02 数据血缘规范

## 1. 血缘定位

- 建立数据**从源到用**的完整传递链路：
  - **广度**：源端表 → ODS → DWD → DIM → DWS → ADS → API / 大屏 / 文件 / 推送外部；
  - **深度**：库 → 表 → 字段；
- 支持**影响分析**：修改上游时下游可见；
- 支持**溯源分析**：质量异常从下游回到源头。

## 2. 节点与边模型

### 2.1 节点类型
- `system`：业务系统（教务/学工 …）
- `database`：数据库/集群
- `schema`：库内 Schema
- `table`：物理表 / 视图 / 物化视图
- `column`：字段
- `api`：数据服务 API
- `dataset`：可视化数据集
- `screen`：大屏页面
- `indicator`：指标
- `tag`：标签
- `script`：SQL/PY 脚本任务
- `pipeline`：ETL DAG
- `collect_job` / `push_job`：采集 / 推送任务
- `file_asset`：文件资产

### 2.2 边类型
- `produced_by`：由任务产生；
- `derived_from`：派生自（字段级 / 表级）；
- `references`：引用（SQL SELECT 中）；
- `joins_with`：JOIN 关系；
- `transforms_to`：字段变换；
- `published_as`：以 API / 指标发布；
- `embedded_in`：被大屏 / 数据集引用。

### 2.3 属性
- 任务：`job_code, run_id, exec_ts`
- 字段边：`transform_expr`（表达式）、`confidence`（0~1）、`extract_method`（sql-parse / manual / ingest）
- 版本：每次变更新增边，旧边标记 `is_latest=false`。

## 3. 血缘来源

| 来源 | 抽取方式 |
| --- | --- |
| 采集任务 | 配置中"源表 → 目标表"直接建立 |
| ETL 设计器 | DAG 即血缘 |
| SQL 任务 | Calcite / JSqlParser 解析 AST，支持 CTE、JOIN、UNION、子查询 |
| Flink SQL | Flink catalog 抽取 |
| 指标定义 | 指标 formula / source_tables 直接读 |
| API 配置 | API 的 SQL 模板解析 |
| 大屏数据集 | dataset config 直接读 |
| 推送任务 | 源 SQL → 外部目标 |
| 外部系统 | 通过 `POST /api/govern/v1/lineage/ingest` 登记 |

## 4. SQL 解析方案

- 基础：Apache Calcite + JSqlParser 组合；
- 方言：PG、CK、Hive 三种；
- 支持：CTE、多层嵌套、UNION、JOIN、CASE WHEN、窗口函数、物化视图；
- 能力边界：
  - 动态 SQL / 拼接 SQL：只能抽出表级；
  - 存储过程 / 自定义函数：手动登记；
- 输出：DAG（node, edge）→ 直接写血缘库。

## 5. 血缘存储（一期 PG）

```sql
-- 见核心表结构清单 bdc_govern.t_lineage_node / t_lineage_edge
```

- 节点主键：`node_key = <type>:<database>.<schema>.<table>.<column>`；
- 边索引：`src_node_id`、`tgt_node_id`；
- 查询：PG 递归 CTE：
```sql
WITH RECURSIVE u AS (
  SELECT e.tgt_node_id AS nid, 1 AS depth
  FROM t_lineage_edge e WHERE e.src_node_id = :start_id
  UNION ALL
  SELECT e.tgt_node_id, u.depth+1
  FROM t_lineage_edge e JOIN u ON e.src_node_id = u.nid
  WHERE u.depth < :max_depth
)
SELECT DISTINCT nid FROM u;
```

二期数据量扩大（节点 > 100 万）切 Nebula Graph。

## 6. 血缘展示

### 6.1 视图
- 表级视图：父节点 = 表，层级 3，折叠字段；
- 字段级视图：展开单表字段，箭头标注转换表达式；
- 影响视图：选定节点 → 高亮所有下游 + 影响任务；
- 溯源视图：选定节点 → 上游 + 产生任务 + 源系统。

### 6.2 技术实现
- 前端：AntV G6；
- 虚拟化：一次绘制 ≤ 500 节点，更多则聚合分组；
- 交互：鼠标悬停显示属性；双击展开；Tab 切换影响/溯源。

## 7. 血缘与变更

### 7.1 变更扫描
- DDL 变更（新增/删除/改类型）→ 变更事件 → 血缘扫描新增字段、检测下游；
- 评估：受影响 API / 指标 / 大屏 数量；
- 自动发起"影响通知"给下游 Owner。

### 7.2 变更审批
- 删除或改类型字段：如果有下游 API / 指标 / 大屏，必须审批；
- 审批流程 `indicator-change` 或 `standard-change`；
- 通过后才允许真正执行 DDL。

## 8. 数据质量与血缘联动

- 质量事件可追溯到上游字段和任务；
- 质量大盘提供"问题源头分析"按钮，跳转血缘图。

## 9. 血缘对外开放

- OpenAPI：`GET /api/govern/v1/lineage/table/{full}` / `column/{full}/{col}`；
- 参数：方向、深度、是否包含字段；
- 权限：至少 `resource:view` 权限。

## 10. 盘点与治理

- 孤儿血缘（无源或无目标）每月扫描并告警；
- 节点属性缺失（无 Owner、无 Description）列入治理待办；
- 年度血缘盘点报告。
