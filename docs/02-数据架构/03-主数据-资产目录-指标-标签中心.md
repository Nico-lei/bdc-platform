# 03 四大中心：主数据 / 资产目录 / 指标 / 标签

## 一、主数据管理（MDM）

### 1. 主数据范围（一期）

| 主数据实体 | 权威来源 | 关键字段 | 同步频率 |
| --- | --- | --- | --- |
| 组织机构 MDM_ORG | 人事处 | org_id, org_code, org_name, parent_id, org_type, level, is_teaching | 实时 |
| 教职工 MDM_STAFF | 人事处 | staff_id, staff_no, id_card_hash, name, gender, birth_date, join_date, status, org_id | 实时 |
| 学生 MDM_STUDENT | 教务处 | student_id, student_no, id_card_hash, name, gender, birth_date, enroll_date, graduate_date, status, major_id, class_id | 实时 |
| 专业 MDM_MAJOR | 教务处 | major_id, major_code, major_name, degree_type, length, college_id | 日 |
| 班级 MDM_CLASS | 教务处 | class_id, class_code, class_name, grade, major_id, tutor_staff_id | 日 |
| 课程 MDM_COURSE | 教务处 | course_id, course_code, course_name, credit, hours, course_type | 日 |
| 校历 MDM_TERM | 教务处 | term_id, academic_year, term_no, begin_date, end_date, week_count | 按学期 |
| 楼栋教室 MDM_ROOM | 资产处 | room_id, room_code, building, floor, capacity | 日 |
| 字典 MDM_DICT | 信息办 | dict_type, dict_code, dict_name, sort, parent_code | 实时 |
| 国标代码 MDM_GB_CODE | 中台 | 民族、国家、行政区划、学科代码 | 年 |

### 2. 主数据治理流程

1. **识别**：数据管理员登记主数据实体与权威源；
2. **清洗**：初始化导入权威源历史数据；
3. **确权**：与业务部门签署主数据权威备忘录；
4. **发布**：发布主数据视图/API；
5. **订阅**：其他系统/其他服务订阅；
6. **变更**：通过 MDM 工作台（管理控制台）变更 → 发布事件 `mdm.sync`；
7. **监督**：数据质量规则每日校验"源数据 ↔ MDM"一致性。

### 3. 主数据服务接口（示例）

```
GET  /api/mdm/v1/staff/{staff_no}             # 单查
POST /api/mdm/v1/staff/query                   # 批量查询
GET  /api/mdm/v1/org/tree                      # 组织树
POST /api/mdm/v1/sync                          # 主动触发同步
WS   /api/mdm/v1/subscribe                     # 订阅变更 (后端内部)
```

### 4. 主数据关键表

详见 `04-数据库详细设计/01-核心表结构清单.md` 中的 `MDM_*` 部分。

### 5. 数据质量保障

- MDM 唯一性：`uk` 约束 + 同步前去重；
- 变更留痕：每次变更写 `t_mdm_change_log`；
- 冲突解决：以权威源覆盖其他来源；冲突事件生成待办由数据管理员介入。

## 二、数据资产目录

### 1. 功能定位

"校内数据超市"的元数据底座：所有数据库、表、字段、API、文件集、指标、标签以统一目录对外展现。

### 2. 资产类型

| 类型 | 描述 | 元数据字段 |
| --- | --- | --- |
| 物理表 | PG/CK/源库中的表 | 库、Schema、表名、字段、分区、Owner |
| 视图/物化视图 | 查询视图 | 同上 |
| API 服务 | 数据服务的 API | Path、Method、参数、响应、鉴权 |
| 文件集 | MinIO 桶中的一组文件 | 桶、前缀、格式、条目数 |
| 指标 | 指标中心指标 | 公式、维度、频度 |
| 标签 | 标签中心标签 | 维度、类型、值域 |
| 数据集 | 经过筛选/脱敏的可共享数据集 | 基于表或 API |

### 3. 资产属性（通用）

- 基本：名称、英文名、责任部门、业务 Owner、技术 Owner；
- 业务：业务域、主题、描述、用途、业务规则；
- 治理：上下架状态、版本、发布时间、是否主数据、是否核心资产；
- 安全：分级（L1~L4）、分类、脱敏策略、访问级别（公开/申请/机密）；
- 质量：质量评分、最近一次质检结果；
- 血缘：上游/下游链接（字段级）；
- 统计：查询次数、订阅数、评分；
- 标签：自由标签、业务标签。

### 4. 目录运营

- 新建资产自动入库（DDL 变更触发）；
- 人工补齐业务属性；
- 上架审批：数据管理员 → 安全管理员；
- 下架：保留档案，标记"已下架"，不再对外可见；
- 定期盘点（季度）。

### 5. 检索

- 关键词全文检索（Elasticsearch），支持中文分词、拼音；
- 多维筛选：业务域、类型、级别、Owner；
- 热门资产、推荐资产（基于使用量 + 协同过滤）。

## 三、指标中心

### 1. 指标体系设计

#### 1.1 指标分类（三层）

1. **原子指标**：最小业务语义单元（如"选课人数"、"课时"），基于单一事实表加简单聚合；
2. **派生指标** = 原子指标 + 时间周期 + 修饰词 + 维度
   - 示例：本学期 + 全日制 + 本科生 + 选课人数；
3. **复合指标**：多个派生指标通过四则运算得到（如"升学率 = 升学人数 / 应届毕业人数"）。

#### 1.2 指标模型

```
指标 Indicator
├── 编码 code (全校唯一，如 IDX_STU_ENROLL_COUNT)
├── 中文名称
├── 业务定义
├── 所属主题域
├── 口径公式 (SQL/脚本模板)
├── 指标类型 (原子/派生/复合)
├── 数据来源 (表/视图/指标)
├── 计算周期 (实时/日/周/月/学期/年)
├── 维度集合 (时间/部门/专业/课程...)
├── 单位
├── 正/负向
├── 精度
├── 预警阈值
├── 责任人 / 责任部门
├── 版本号 / 生效期
├── 上下架状态
└── 业务标签
```

#### 1.3 指标生产

- **离线**：DS 调度 SQL 任务写入 `ads_*` 表；
- **实时**：Flink 作业写 ClickHouse `ads_rt_*`；
- **即查**：根据指标定义动态拼 SQL，缓存到 Redis（10min）。

#### 1.4 指标 API

```
GET /api/indicator/v1/{code}?period=2026&dim.org=xxx
POST /api/indicator/v1/batch  # 批量
GET /api/indicator/v1/explore # 维度下钻
```

#### 1.5 指标治理

- 每个派生/复合指标必须引用已登记的原子指标；
- 公式变更走审批 + 版本号；旧版本保留 1 年；
- 指标血缘 → 数据血缘贯通；
- 每日校验指标值与业务系统差异报警。

#### 1.6 示例指标（预置）

| 编码 | 中文名 | 口径 |
| --- | --- | --- |
| IDX_STU_ENROLL | 在读学生数 | `dim_person` 在读 count |
| IDX_STU_NEW_CNT | 当年新生数 | 当学年入学 |
| IDX_STU_GRAD_RATE | 毕业率 | 毕业生 / 应届 |
| IDX_TEACH_COURSE_CNT | 开课门数 | `dwd_teach_course` distinct |
| IDX_TEACH_SELECT_CNT | 学生选课人次 | `dwd_teach_course_select` count |
| IDX_TEACH_AVG_SCORE | 平均成绩 | `dwd_teach_score` avg |
| IDX_FIN_INCOME | 收入总额 | `dwd_fin_income` sum |
| IDX_LIB_BORROW_CNT | 借阅量 | `dwd_lib_borrow` count |
| … | … | … |

## 四、标签中心

### 1. 标签分类

| 类型 | 生成方式 | 示例 |
| --- | --- | --- |
| 规则标签 | SQL/表达式，从业务数据生成 | `stu.是否家庭经济困难` = 资助档案有记录 |
| 统计标签 | 基于指标统计+阈值划分 | `stu.学业水平` = A/B/C（按 GPA 分段） |
| 算法标签 | 基于算法（一期不强制） | `stu.辍学风险` = 模型输出 |

### 2. 标签对象

- 主数据对象：学生、教职工、部门、课程、资产…；
- 多对多：一个对象可绑定多个标签。

### 3. 标签模型

```
标签 Tag
├── code
├── 中文名
├── 对象类型 (stu/staff/course...)
├── 值类型 (枚举/数值/布尔)
├── 值域
├── 定义规则 (SQL/表达式)
├── 刷新周期
├── 所属体系
├── 责任人
├── 分级 (L1~L4)
├── 上下架状态
└── 业务标签
```

### 4. 标签生产

- 规则/统计：DS 定时任务 → 写 `bdc_tag.t_tag_value_<object>`；
- 算法：二期预留接入点；
- 统一查询 API：`GET /api/tag/v1/object/{type}/{id}`。

### 5. 画像使用

- 画像宽表（示例：`dws_stu_profile_w`）
  - 主键：`student_id`
  - 字段：若干基础属性 + 标签列（稀疏 JSONB）；
- 大屏/应用按学生画像进行群组筛选、分布统计。

### 6. 隐私 & 合规

- 标签继承对象的分级；
- 画像对外 API 强制脱敏；
- 算法标签需申请备案，保留训练数据与模型版本；
- 师生可申请查询自己的标签（个保法权利）。
