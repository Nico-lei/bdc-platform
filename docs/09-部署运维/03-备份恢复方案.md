# 03 备份与恢复方案

## 1. 备份对象与策略

| 对象 | 工具 | 频率 | 保留 | RPO | RTO |
| --- | --- | --- | --- | --- | --- |
| PostgreSQL 业务库 | pgBackRest | 全:日02:00；增:4h | 30/12/12/3 | ≤ 15min | ≤ 1h |
| PostgreSQL 数仓 | pgBackRest | 全:日02:00；差:12:00；WAL 1min | 30/12/12/3 | ≤ 5min | ≤ 2h |
| ClickHouse DWS/ADS | clickhouse-backup | 全:周；增:日 | 12/3 | ≤ 24h | ≤ 2h |
| MinIO 对象 | mc mirror → 冷备集群 | 实时+每日对账 | 同源+版本 30d | ≤ 5min | ≤ 30min |
| Kafka | Topic 配置导出 + 消费位点备份（学校具备条件时可扩展 MirrorMaker2） | 日 | 7 日保留 | ≤ 24h | 小时级 |
| Nacos 配置 | 每日导出 | 日 | 30 | ≤ 24h | ≤ 10min |
| Nacos/Flowable DB | pgBackRest | 日 | 30 | ≤ 15min | ≤ 30min |
| DS 元库 | pgBackRest | 日 | 30 | ≤ 15min | ≤ 30min |
| Harbor 镜像 | 定期同步到冷备 | 日 | 30 | 快照 | 1~2h |
| 源代码 | 代码仓库镜像备份 + NAS 备份 | 日 | 30 | 天级 | 小时级 |
| Elasticsearch 审计索引 | snapshot → MinIO | 日 | 30 | 天级 | 1h |

## 2. 备份介质

- **主目标**：MinIO `bdc-backup` 桶（独立集群或独立节点组）；
- **冷备目标**：学校自备 NAS 或第二套 MinIO；
- **长期归档**：磁带/蓝光（由学校机房决定，本中台输出 Parquet 文件到归档桶）。

## 3. pgBackRest 配置要点

```ini
[global]
repo1-type=s3
repo1-s3-endpoint=minio-backup:9000
repo1-s3-bucket=bdc-backup
repo1-s3-region=us-east-1
repo1-s3-verify-tls=y
repo1-retention-full=30
repo1-retention-diff=12
repo1-cipher-type=aes-256-cbc         # 可替换为国密 SM4（BabaSSL 构建）
repo1-cipher-pass=<KMS 注入>
process-max=4
compress-type=zst
compress-level=3

[pg-main]
pg1-host=pg-master
pg1-port=5432
pg1-user=postgres
```

命令：
- 全量：`pgbackrest --stanza=pg-main backup --type=full`
- 增量：`--type=incr`
- WAL 归档：PG 侧 `archive_command = pgbackrest --stanza=pg-main archive-push %p`
- PITR 恢复：`pgbackrest --stanza=pg-main --type=time --target='2026-04-20 08:30:00' restore`

## 4. ClickHouse 备份

- 工具：`clickhouse-backup` + S3（MinIO）；
- 备份粒度：按 DB（`bdc_dw_olap`）+ 按表（大表单独）；
- 恢复：`clickhouse-backup restore <backup_name>`，支持增量；
- 验证：季度抽样恢复到隔离实例。

## 5. MinIO

- **版本化** + **生命周期** 策略：
  - 版本保留 30 天软删；
  - 老版本 90 天后转冷；
  - 30 天之外的备份 Lifecycle 自动删除；
- **复制**：`mc replicate` 实时 + `mc mirror --overwrite` 兜底校验；
- **审计**：开启 Audit Webhook 发送到审计服务。

## 6. 恢复演练

| 场景 | 演练频率 | 流程 |
| --- | --- | --- |
| PG 全库恢复（新实例） | 季度 | 拷贝备份 → 在线恢复 → 校验数据一致性 |
| PG PITR 到昨日 08:30 | 季度 | 验证 WAL 归档完整性 |
| CK 表恢复 | 季度 | 10 个代表表 |
| MinIO 误删 | 季度 | 从版本恢复 + 从冷备同步 |
| 多对象并发恢复 | 年度 | 模拟灾难 |

## 7. 回滚（应用 & DB）

- 应用：回切上一版镜像 tag 并重新执行 `docker compose up -d`；
- DB：
  - 有 Flyway 回滚脚本使用脚本；
  - 无脚本则基于备份选择时间点 PITR；
- 敏感变更：部署前手动 `pg_dump` 快照。

## 8. 安全

- 备份加密：AES-256 / SM4（依构建版本）；
- 传输加密：TLS；
- 密钥：KMS 管理，备份与生产密钥分离；
- 访问：只有 `backup` 服务账号可写；恢复操作需审计管理员 + 安全管理员双签（高敏）。

## 9. 监控指标

- 每日备份成功率 100%，失败即告警；
- 备份时长不超过窗口 2 h；
- 备份大小波动 > 30% 告警；
- 恢复演练成功率 100%。

## 10. 归档（长期保留）

- PG 老分区（> 2 年）：pg_partman 自动脱离 → `pg_dump` → MinIO 归档桶 + 国密压缩；
- CK 老分区（> 5 年）：`MOVE PARTITION` 到冷卷 → 或 `BACKUP` 到 MinIO；
- 审计日志：每月打包 Parquet → MinIO 归档，保留 ≥ 2 年；
- 所有归档目录建立索引表，供审计检索。

## 11. 文档化

- 每次备份/恢复/演练记录到 `bdc_audit.t_backup_log`；
- 年度报告：备份覆盖率、恢复成功率、平均 RTO/RPO；
- 书面预案：随《应急预案》一同上报学校信息办。
