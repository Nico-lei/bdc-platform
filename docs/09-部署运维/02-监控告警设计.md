# 02 监控告警设计

## 1. 监控体系

- **指标**：Prometheus + Grafana（业务/基础/中间件/JVM）
- **日志**：Loki + Promtail + Grafana 日志界面
- **链路**：SkyWalking 9（Agent 注入到 JVM）
- **应用埋点**：Micrometer（Spring Boot Actuator）
- **告警**：Alertmanager → 钉钉/企微/邮件/短信

## 2. 监控分层

| 层 | 对象 | 指标示例 |
| --- | --- | --- |
| 基础设施 | 节点 | CPU/Mem/Disk/Net/LoadAvg |
| 容器 | Docker Container | CPU/Mem/IO、重启次数 |
| 中间件 | PG | QPS、慢查询、连接数、WAL 堆积、主备延迟 |
|  | CK | 查询 QPS、MergeTree part 数、内存、磁盘 |
|  | Kafka | Topic LAG、吞吐、ISR |
|  | MinIO | API QPS、磁盘利用率 |
|  | Redis | CPU、内存、命中率、连接 |
|  | Elasticsearch | JVM GC、查询延迟、热点索引 |
|  | Flink | checkpoint、back pressure、延迟 |
|  | DS | 工作流状态、任务堆积 |
| 应用 | 微服务 | HTTP QPS、P95/P99、错误率、JVM |
| 业务 | 大盘 | 采集成功率、质量分、ETL 完成率、API 调用量、下载量 |

## 4. 大盘设计（Grafana）

| 大盘 | 主要内容 |
| --- | --- |
| 平台总览 | 节点/服务/中间件健康一屏 |
| 微服务总览 | HTTP、错误率、JVM、TopN |
| 采集监控 | 任务成功率、CDC 延迟、错误记录 |
| 治理监控 | DQ 评分趋势、事件 TopN、ETL 耗时 |
| 数据服务 | QPS、P95、TopN API、错误 |
| 大屏监控 | 在线大屏、渲染耗时、数据集刷新 |
| 审计态势 | 登录失败、批量下载、KMS 调用 |
| 数据库健康 | PG、CK、Redis、Elasticsearch |
| Kafka / Flink | LAG、吞吐、Back Pressure |
| 备份与容量 | 备份成功率、磁盘利用 |

## 5. 日志规范

- 统一 JSON 行（`%d{ISO8601} %level [%t] [trace=%X{traceId}] %logger{36} - %msg%n`）；
- 重要模块使用**结构化日志**（key=value）；
- 日志量：普通微服务 < 200 MB/day；业务高峰可扩容；
- 保留：Loki 热存 30 天 + 归档 MinIO 2 年；
- 查询：Grafana Explore / LogQL。

## 6. 链路追踪

- SkyWalking Agent 通过基础镜像预置或启动参数挂载方式注入；
- 采样率：默认 100%，高峰期可降至 20%；
- 贯穿：Gateway → 服务 → OpenFeign → 中间件（JDBC、Redis、Kafka）；
- 业务自定义 Tag：`userId`、`orgId`、`apiCode`（不含敏感字段）。

## 7. 告警策略

### 7.1 告警级别
- P0：服务全量不可用、数据库主节点宕、国密密钥异常；
- P1：主要服务错误率/耗时超阈值；
- P2：次要服务或单个组件劣化；
- P3：资源使用率预警、即将触发。

### 7.2 告警规则（示例）
```
# 容器重启
ALERT container_restart_high IF increase(container_restart_total[5m]) > 0
# HTTP 错误率
ALERT api_5xx_high IF sum(rate(http_server_requests_total{status=~"5.."}[5m])) / sum(rate(http_server_requests_total[5m])) > 0.02 FOR 5m
# 数据库复制延迟
ALERT pg_replication_lag IF pg_stat_replication_lag_seconds > 30
# Kafka LAG
ALERT kafka_lag_high IF kafka_consumer_lag > 50000 FOR 10m
# 磁盘
ALERT disk_usage_critical IF node_filesystem_avail_bytes / node_filesystem_size_bytes < 0.1
# 审计异常
ALERT audit_failed_login IF increase(bdc_auth_login_fail_total[10m]) > 50
# 批量下载
ALERT market_bulk_download IF increase(bdc_market_download_total{user=~".+"}[5m]) > 100
# 质量评分跌落
ALERT dq_score_drop IF (bdc_dq_score < 80) AND (changes(bdc_dq_score[1h]) > 0)
```

### 7.3 通知渠道
- P0：短信 + 电话（第三方集成）+ 钉钉 + 邮件；
- P1：钉钉 + 邮件；
- P2：钉钉；
- P3：平台消息 + 日报。

### 7.4 值班
- 7×24 值班（工作日白班 + 夜间 On-Call）；
- 上报 & 升级路径；
- Runbook：每条告警附处置指南链接；
- 事后 RCA。

## 8. SLO 与 SLI

| 服务 | SLO | 度量 |
| --- | --- | --- |
| 数据服务 API | P95 ≤ 50ms，成功率 ≥ 99.9% | 90 天滚动 |
| 平台整体 | 可用性 ≥ 99.9% | 服务不可用时间累计 |
| 大屏首屏 | ≤ 3s | 埋点上报 |
| 采集 CDC 延迟 | ≤ 5s | Flink metric |
| 批处理窗口完成率 | ≥ 99% | 工作流 SLA |
| 告警响应 | P0 ≤ 15min | 人工 + 平台 |

## 9. 演练与应急

- 半年一次故障演练（通过故障注入脚本或人工演练）；
- 备份恢复演练季度一次；
- 切换 / 切流演练；
- 应急联系人手册 + 一键回滚脚本。
