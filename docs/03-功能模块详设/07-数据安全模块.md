# 07 数据安全模块详设

## 1. 模块职责

- 为全平台提供**分类分级、权限控制、脱敏、加密、水印、审计**底座；
- 满足**等保三级**与《数据安全法》《个人信息保护法》要求。

## 2. 能力矩阵

| 能力 | 覆盖粒度 |
| --- | --- |
| 权限控制 | 库 / 表 / 行 / 列 |
| 分类分级 | 资产 / 字段 自动识别 + 人工复核 |
| 脱敏 | 静态 + 动态 |
| 加密 | 传输 + 存储 + 字段级 + 备份 |
| 水印 | 下载可见 + 隐写 |
| 审计 | 系统 / 操作 / 数据访问 / SQL |
| 三权分立 | 系统 / 安全 / 审计 |

## 3. 权限模型（RBAC + ABAC 混合）

### 3.1 RBAC 基础

- 组织 Org（树状） + 用户 User + 角色 Role；
- 用户可隶属多组织；角色绑定菜单 + 操作权限；
- 数据权限与菜单权限分离：
  - 菜单权限：`t_role_menu`；
  - 数据权限：`t_role_data_scope` + `t_row_rule` + `t_col_rule`。

### 3.2 数据权限规则

按对象绑定：
- **库/表级**：角色/用户 对 Schema/表 是否可见；
- **行级**：SQL WHERE 模板（含占位符），运行时渲染
  - 示例：`dept_id = :current_dept_id`、`grade_id IN :current_grades`；
- **列级**：可见/脱敏/密文/不可见；
- **时段**：仅允许某时间窗口访问；
- **来源**：IP 白名单；
- **条件上下文**：环境属性（来源应用、终端类型）。

### 3.3 ABAC 增强

- 属性：用户属性（角色、岗位、级别）+ 资源属性（分级、Owner、域）+ 环境属性（时间、IP、MFA 状态）；
- 引擎：内置简化 XACML，支持策略表达式；
- 典型：当 `user.dept = resource.owner_dept AND resource.level <= L2 THEN allow`。

### 3.4 授权粒度

- 全局角色权限：由 sysadmin 维护；
- 数据权限：由数据管理员 + 部门数据管理员维护；
- 运行态：鉴权中心（bdc-uac）内置缓存，P99 < 10ms。

## 4. 分类分级

### 4.1 四级分类

见 `00-项目概述/02-核心假设与选型决策.md#E` 已定义。

### 4.2 自动识别规则

内置正则与字典：
- 身份证、手机、邮箱、银行卡、军官证、护照；
- 姓名（结合字典）；
- 工资、薪资、绩效、社保、公积金、奖惩；
- 成绩、考评、档案；
- 经纬度/地址；
- 健康、病历、基因；
- 密码/Hash/Token/密钥。

### 4.3 识别流程

- 每日自动扫描新表/变更字段；
- 命中规则 → 打分 → 建议分级 → 待复核；
- 人工复核 2 人通过方生效；
- 所有变更留痕。

### 4.4 分级生效

- 写入 `bdc_security.t_col_class`：字段 → 级别；
- 事件 `security.class.change` 广播；
- 各业务服务订阅，刷新本地缓存；
- 鉴权 / 脱敏 / 审计 均读取该表。

## 5. 脱敏

### 5.1 两种形态

- **静态脱敏**：数据落地 DWD 前执行，源表保留原值；
- **动态脱敏**：读取时按用户权限动态脱敏，存储原值。

### 5.2 脱敏算法（内置）

| 算法 | 效果示例 |
| --- | --- |
| MASK_NAME | 张*三 |
| MASK_MOBILE | 138\*\*\*\*1234 |
| MASK_IDCARD | 110\*\*\*\*\*\*\*\*\*\*\*\*5678 |
| MASK_EMAIL | m\*\*\*@edu.cn |
| MASK_ADDR | 北京市\*\*\*\*区 |
| MASK_BANK | 6222\*\*\*\*\*\*\*\*1234 |
| HASH_SM3 | 摘要 |
| RANDOM | 同类型随机值（保持分布） |
| DATE_SHIFT | 日期平移 ±30d |
| RANGE | 数值四舍五入到百/千 |
| TRUNCATE_N | 截断 N 字符 |
| ENCRYPT_SM4 | 返回密文（仅高级场景） |

### 5.3 规则与优先级

- 规则基于字段 + 用户角色 + 环境：
  - 若用户未授权 → 按字段默认脱敏；
  - 已授权 → 按授权级别（可能仍带脱敏）；
- 规则命中顺序：精确用户 > 角色 > 部门 > 全局默认；
- 规则热加载（Nacos 配置刷新 + 事件推送）。

### 5.4 实现

- 对 API 返回：`@Desensitize` 注解在 DTO 字段 + AOP 处理；
- 对 SQL 查询：`bdc-common-security` 重写 SELECT（ShardingSphere-Proxy 可选）；
- 对大屏查询：在 ADS 物化视图层提前脱敏；
- 对下载文件：导出组件注入脱敏；
- 对文件预览：渲染层脱敏（敏感区打码或文本替换）。

## 6. 加密

### 6.1 分层密钥

- **MK 主密钥**：存储于 HSM / KMS（软 HSM 启动时加密保护）；
- **DEK 数据加密密钥**：按业务域派生；
- **FEK 字段密钥**：按 `(table, column)` 派生；
- 密钥轮换：MK 年度、DEK 半年、FEK 按需；
- 轮换不影响历史数据读取（保留旧版本）。

### 6.2 算法

| 场景 | 算法 |
| --- | --- |
| 传输 | 国密 TLS（SM2 + SM3 + SM4） / 备选 TLS1.3 |
| 存储（字段级） | SM4 GCM（推荐）/ CBC |
| 存储（磁盘） | OS 层 LUKS（国密 SM4 可选） |
| Hash / 摘要 | SM3 |
| 签名 | SM2 |
| 备份 | SM4 加密 + 访问控制 |

### 6.3 字段加密实现

- 注解：`@EncryptField(alg="SM4", keyId="FEK_STU_IDCARD")`；
- 查询：
  - 精确：基于 Hash 副列查找；
  - 模糊：不支持（合规要求），需走审批获得明文视图。

### 6.4 密钥管理 API

```
POST /api/security/v1/kms/dek            # 创建 DEK
POST /api/security/v1/kms/dek/{id}/rotate
POST /api/security/v1/kms/envelope/encrypt
POST /api/security/v1/kms/envelope/decrypt
```
三权分立：安全管理员管理 KMS；密钥材料不对业务透明。

## 7. 水印

### 7.1 可见水印

- 前端 Web：平铺水印（姓名 + 工号 + 时间，半透明灰）；
- 下载文件：
  - CSV：前缀注释行；
  - Excel：背景 + 页脚；
  - PDF：跨页平铺；
  - 图片：半透明文字；
  - 视频：视频流嵌入或播放器覆盖；
- 大屏：左下角默认水印，管理员可配置。

### 7.2 隐写水印

- 图片：LSB + 频域；
- PDF/Word：隐藏字（白底白字微量）+ 元数据字段；
- CSV：行重排序微扰动（保留业务正确）；
- 二次溯源：若外泄，上传样本 → 水印解码出下载用户。

## 8. 审计

### 8.1 审计对象

- 登录、登出、登录失败；
- 权限变更（授权/回收）；
- 数据查看、下载、导出、推送；
- SQL 执行（高危语句）；
- 配置修改（安全 / 分级 / 规则）；
- KMS 操作；
- 审批流节点流转；
- 系统启停、运维操作。

### 8.2 审计管道

```
服务 → Kafka(audit.*) → bdc-audit 消费 → 写 PG 主库 + Elasticsearch 索引 + 冷归档 MinIO Parquet
```

### 8.3 审计特性

- 日志**不可删除**、**不可修改**（PG 表设置只写，触发器阻止更新）；
- 完整性：每批日志计算 SM3 Hash 链（前一条 Hash 纳入下一条），周期签名（SM2）存档；
- 保留：**≥ 6 个月可查**；**≥ 2 年归档**（等保要求）；
- 审计管理员专有查询面；
- 告警：异常登录、批量下载、敏感 SQL。

## 9. 三权分立

| 管理员 | 职责 | 禁止 |
| --- | --- | --- |
| 系统管理员 sysadmin | 系统配置、部署、节点 | 接触审计日志、密钥、数据授权 |
| 安全管理员 secadmin | 权限、分级、脱敏、加密、审批 | 操作系统底层、接触审计 |
| 审计管理员 auditadmin | 审计 | 修改配置、授权、接触密钥 |

三权账号之间**禁止兼任**；任意管理员动作写审计；关键动作双人复核（KMS 轮换、审计日志导出）。

## 12. 安全自测与合规

- 定期（月度）运行自动化安全扫描（配置 / 弱口令 / 权限矩阵）；
- 季度渗透测试（外部或内部红队）；
- 年度第三方等保测评；
- DLP 旁路监控可选。
