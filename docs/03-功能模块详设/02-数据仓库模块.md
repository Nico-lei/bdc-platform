# 02 数据仓库模块详设

## 1. 模块职责

- 承载结构化分层存储（ODS/DWD/DIM/DWS/ADS）与非结构化对象存储；
- 提供仓库元数据视图（层级、表、字段、分区、容量）；
- 数据持久化、生命周期、备份恢复。

## 2. 存储物理分布

| 层 | 存储 | 库/集群 |
| --- | --- | --- |
| ODS | PostgreSQL 15 | `bdc_dw_pg` 实例 |
| DWD | PostgreSQL 15 | `bdc_dw_pg` 实例（同上） |
| DIM | PostgreSQL 15 | `bdc_dw_pg` 实例 |
| DWS | ClickHouse 24.3 | `bdc_dw_ck` 集群（2 分片 × 2 副本） |
| ADS | ClickHouse 24.3 | `bdc_dw_ck` 集群 |
| 非结构化 | MinIO | `bdc-files-*` 桶 |

## 3. 结构化分层详设

### 3.1 ODS

- Schema 按源系统分：`ods_jwxt`, `ods_xgxt`, `ods_hr`, ...
- 默认普通表；仅超大 ODS 表按月分区；
- 每张 ODS 表配套视图 `v_<table>_latest` 返回最新快照；
- 索引：主键 + `_etl_ts` + 业务常用查询列；
- 容量：按"日增 10G"测算，年 3.6T（压缩前），一期优先通过索引、冷热归档和少量月分区控制体量，不采用日分区细拆。

### 3.2 DWD

- 按主题域分 Schema：`dwd_stu`, `dwd_teach`, `dwd_fin` 等；
- 事实表（Transactional Fact）、周期快照（Periodic Snapshot）、累积快照（Accumulating Snapshot）三类；
- 字段规范化：类型、编码、空值、长度；
- 代理键：以 `<entity>_sk` 关联 DIM 维度（BIGINT，DIM 表生成）；
- 写入方式：SQL 批处理（DolphinScheduler + PL/pgSQL 或 Flink SQL Sink JDBC）；
- 分区：默认不分区；高增长事实表按月分区，避免与 ODS 做一一对应的细粒度分区；
- 业务键 + 生效时间 唯一索引。

### 3.3 DIM

- Schema：`dim`；
- 代理键生成：`GENERATED ALWAYS AS IDENTITY`（PG 10+）；
- SCD2 拉链字段：`effect_from`、`effect_to`（默认 9999-12-31）、`is_current`；
- 慢变维度刷新：每日凌晨从 MDM 增量更新；
- 历史表不做物理删除，只标 `is_current=false`。

### 3.4 DWS

- ClickHouse `ReplicatedAggregatingMergeTree` 或 `ReplicatedMergeTree`；
- 主题宽表：`dws_stu_profile_w`、`dws_teach_class_m`、`dws_fin_dept_d`…
- 分区：`toYYYYMM(stat_date)`；
- 排序键：常用过滤维度（如 `dept_id, grade, stat_date`）；
- TTL：默认 5 年，逾期 `TO VOLUME 'cold'`；
- 物化视图（MV）承载实时聚合。

### 3.5 ADS

- 面向场景：大屏 / BI / API；
- 字段少、聚合预计算充分；
- 命名 `ads_<场景>_<粒度>`；
- 更新频率：多数 T+1；实时场景用 `ads_rt_*`；
- 保留周期：5 年 + 人工决定是否归档。

## 4. 非结构化存储

### 4.1 桶划分

| 桶名 | 用途 | 分级 | 版本化 | 加密 |
| --- | --- | --- | --- | --- |
| `bdc-files-public` | 公共资源（公告图片等） | L1 | off | 否 |
| `bdc-files-internal` | 内部文档 | L2 | on | SSE-SM4 |
| `bdc-files-sensitive` | 敏感档案、身份证扫描 | L3 | on | SSE-SM4 |
| `bdc-files-confidential` | 机密（体检、心理、纪检档案） | L4 | on | SSE-SM4 + 双因素审批下载 |
| `bdc-backup` | 备份 | 内部 | on | SSE-SM4 |
| `bdc-datax-tmp` | 采集/推送临时 | 内部 | off | 否 |
| `bdc-logs` | 原始日志归档 | 内部 | on | SSE-SM4 |

### 4.2 对象命名

```
<bucket>/<domain>/<year>/<month>/<day>/<uuid_or_bizId>.<ext>
```

### 4.3 元数据表

`bdc_warehouse.t_file_asset`：
- `id`（UUID）
- `bucket`, `object_key`
- `file_name`, `file_size`, `mime_type`
- `hash_sm3`, `hash_sha256`
- `biz_domain`, `biz_id`, `biz_meta`（JSONB）
- `class_level`（L1-L4）
- `owner_org_id`, `owner_user_id`
- `upload_time`, `last_access_time`
- `retention_policy`, `expire_time`
- `is_encrypted`, `encrypt_alg`, `dek_id`
- `is_deleted`, `delete_time`

### 4.4 上传/下载流程

```
应用 ──(预签名请求)──▶ bdc-file
                         │
                         ▼
                    生成 presigned PUT URL（短时有效、SM4 SSE 头）
                         │
                         ▼
                    写入 MinIO
                         │
                         ▼
                    回调 bdc-warehouse 登记元数据
                         │
                         ▼
                    审计记录（bdc-audit）
```

下载：
- 预签名 GET URL（短时 TTL 5 min）；
- 下载前检查权限（bdc-uac）；
- 敏感/机密文件：服务端代理下载 + 嵌入可见水印（姓名 + 工号 + 时间） + 隐写水印（用户ID 哈希）；
- 审计记录下载行为。

## 5. 数据持久化保障

- **WAL 归档**：PG 15 WAL 每 1 min 归档到 MinIO 备份桶；
- **副本**：PG 主备流复制；CK 2 副本；MinIO EC 4+2；
- **校验**：每日 2:00 全量校验（PG `pg_checksums`、CK `CHECK TABLE`、MinIO `mc mirror` 对账）；
- **容量告警**：磁盘 70% Warning、85% Critical，触发自动归档；
- **扩容预案**：PG 使用 LVM 可在线扩展；CK 扩分片需重平衡（脚本化）；MinIO 可新增节点扩容（重平衡自动）。

## 6. 备份与恢复

### 6.1 PostgreSQL

- 工具：**pgBackRest**（开源，Apache 2.0）；
- 全量：每日 02:00；增量：每 4h；差异备份：每日中午；
- 备份存储：MinIO `bdc-backup` 桶；
- 保留策略：日备 30、周备 12、月备 12、年备 3；
- 恢复演练：季度一次，演练到**非生产库 + 指定时间点 PITR**。

### 6.2 ClickHouse

- 工具：**clickhouse-backup**；
- 策略：每日全量（大表改为增量）；
- 存储：MinIO `bdc-backup`；
- 恢复测试：季度一次。

### 6.3 MinIO

- 工具：`mc mirror --watch` 到二套 MinIO（冷备实例）；
- 版本化开启 + 保留 30 天；
- Lifecycle：
  - 90 天转低成本 EC 6+2；
  - 5 年后移冷或删除（按分级策略）。

### 6.4 业务服务库

- 同 PG 策略，但备份力度更轻：每日一次全备；
- Flyway 脚本与 git 版本绑定，可快速重建。

## 7. 冷热分层

| 分层 | 热 | 温 | 冷 |
| --- | --- | --- | --- |
| PG | `pg_default` | `pg_warm`（普通 HDD 表空间） | `pg_cold`（归档 Parquet/Extern ts） |
| CK | SSD `hot` 卷 | SSD `warm` | HDD `cold` / MinIO `s3_cold` |
| MinIO | 默认 EC 4+2 | EC 6+2 | Glacier-like（本地归档桶） |

## 8. 仓库元数据视图（管理控制台）

- 分层树（ODS → ADS）+ 非结构化桶树；
- 每张表：结构、行数、大小、最后更新、责任人、血缘、质量评分、分级分类、是否已上架；
- 非结构化：桶 → 子前缀 → 文件统计（数量 / 总大小 / 过期数量）；
- 一键操作：查看样本（Top 100）、导出 DDL、备份、恢复（仅审计管理员可触发）。

## 9. 非功能指标

| 项 | 目标 |
| --- | --- |
| PG 单 OLTP 写 | 5000 TPS |
| PG 分析查询 P95 | 2s（明细）/ 10s（复杂 join） |
| CK 分析查询 P95 | 500ms（标准 DWS） |
| MinIO 吞吐 | 200MB/s 持续 |
| 备份恢复 RTO | ≤ 2 h |
| 备份 RPO | ≤ 15 min |

## 10. 接口（摘要）

```

GET /api/warehouse/v1/tables?layer=ods
GET /api/warehouse/v1/table/{fullName}     # 详情
GET /api/warehouse/v1/table/{fullName}/sample?limit=100
GET /api/warehouse/v1/table/{fullName}/ddl
POST /api/warehouse/v1/file/presign/upload
POST /api/warehouse/v1/file/presign/download
GET /api/warehouse/v1/file/{id}
POST /api/warehouse/v1/backup/run          # 管理员手动触发
GET /api/warehouse/v1/backup/history
POST /api/warehouse/v1/restore/{backupId}  # 需审计管理员 + 安全管理员双签
```
