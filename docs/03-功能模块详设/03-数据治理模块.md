# 03 数据治理模块详设

## 1. 模块职责

- 构建数据质量、数据标准、ETL 编排、任务调度、血缘追踪一体化数据治理体系；
- 支撑批量 + 精细化 + 实时 + 定时 的数据处理能力。

## 2. 核心能力

1. 数据质量管理（DQ）
2. 数据标准管理（Data Standard）
3. 数据转换清洗（ETL 规则、清洗函数库）
4. 可视化 ETL 设计器（拖拽）
5. SQL / Python / Shell 脚本任务
6. 任务调度 + 依赖 + 重跑 + 补数 + 告警
7. 数据血缘（库 / 表 / 字段 三级）
8. 数据地图

## 4. 数据质量管理（DQ）

### 4.1 六大维度 + 预置规则

| 维度 | 说明 | 预置规则 |
| --- | --- | --- |
| 完整性 | 是否缺失 | 空值率 / 必填校验 / 关键字段缺失 |
| 准确性 | 是否正确 | 取值范围 / 正则 / 字典匹配 / 跨字段一致 |
| 唯一性 | 是否重复 | 主键唯一 / 业务键去重 / 组合唯一 |
| 一致性 | 跨源是否一致 | 跨表/跨库对账 / 主数据一致性 |
| 及时性 | 是否延迟 | 数据最新时间 ≤ 阈值 / 任务延迟 |
| 有效性 | 是否合规 | 身份证/手机号/邮箱格式 / 编码合规 |

### 4.2 规则模型

```
DQ 规则 Rule
├── code
├── name
├── 维度 (6维)
├── 目标 (表/字段/跨表)
├── 规则表达式 (SQL/模板/内置函数)
├── 阈值 (Critical/Warning)
├── 执行周期 (实时/小时/日/手动)
├── 责任部门 / 责任人
├── 告警方式 (消息/邮件/短信/钉钉)
├── 是否阻断下游 (硬阻断/软提醒)
├── 版本 / 生效期
└── 状态
```

### 4.3 执行引擎

- 规则转 SQL → 执行 → 返回命中数与抽样；
- 实时规则：Flink SQL 作业 → 命中写 `bdc_govern.t_dq_incident`；
- 离线规则：DS 调度 + bdc-govern 执行；
- 结果汇总：`t_dq_result`（表级质量评分 0~100）；
- 事件：`govern.quality` 事件广播给 metadata/blueBoard 实时展示。

### 4.4 DQ 报表

- 部门级质量大盘；
- 主题域质量分排名；
- 近 30 天趋势；
- Top10 问题表；
- 整改跟踪。

## 5. 数据标准

### 5.1 标准类型

| 类型 | 示例 |
| --- | --- |
| 业务标准 | 指标口径、业务术语 |
| 技术标准 | 字段命名、类型规范 |
| 代码标准 | 国标代码（民族、国家）、校标代码 |
| 管理标准 | 分级分类、生命周期 |

### 5.2 执行链路

- 数据标准 → 字段映射 → 采集/ETL 时自动应用 → 违反进入质量事件。

## 6. ETL 可视化设计器

### 6.1 前端实现

基于 **AntV X6**，提供拖拽画布。节点分类：

| 类别 | 节点 |
| --- | --- |
| 输入 | 读表（PG/CK）、读文件、读 Kafka、读 API |
| 转换 | 过滤、列映射、SQL 片段、分组聚合、Join、Union、窗口、JSON 解析、脱敏、加密、字典映射、日期格式化、正则提取、自定义 Python |
| 输出 | 写表（PG/CK）、写文件、写 Kafka、写 API |
| 条件 | 条件分支、错误分流、合并 |
| 控制 | 开始、结束、参数 |

### 6.2 后端生成

- 设计器保存为 `pipeline.json`（DAG）；
- 后端将 DAG 翻译为：
  - **简单线性**：SQL 模板（PG SQL 或 CK SQL）；
  - **复杂 / 多源**：Flink SQL 作业；
  - **脚本节点**：Python（Docker 沙箱） / Shell；
- 提交到 DS 或 Flink；
- 保留源图与执行计划（便于 Debug / 血缘抽取）。

### 6.3 预览 / 调试

- 任意节点可 "预览样本 100 行"；
- 支持 Mock 数据；
- 错误在节点上以红色标注并列出异常信息。

## 7. 脚本任务

### 7.1 SQL 任务

- 编辑器：Monaco Editor + SQL 关键字提示 + 表/字段补全（从元数据中心拉）；
- 支持多种目标引擎：PG / CK / Oracle / DM / Hive（二期）；
- 参数化：`${biz_date}`、`${batch_id}`、自定义参数；
- 事务：单任务内 PG BEGIN/COMMIT，出错自动回滚；CK 任务无事务但支持写入缓冲。

### 7.2 Python 任务

- 执行环境：Docker 镜像 `bdc/python-runtime:3.11`（内置 pandas/numpy/requests/psycopg2/clickhouse-driver/pyspark可选/national-crypto）；
- 隔离：容器、CPU/内存限制、无网络出互联网（仅限白名单）；
- 传参：`sys.argv` + 环境变量；
- 输出：stdout/stderr → 日志中心；
- 安全：禁止 `subprocess`（除非白名单）、禁止 `os.system`；静态扫描 `bandit`。

### 7.3 Shell 任务

- 允许：数据搬运、文件清理、调用标准 CLI；
- 禁止：root、rm -rf 敏感路径；
- 执行账号 `bdc-runner`，最小权限。

## 8. 任务调度

基于 DolphinScheduler（详见 `07-任务编排/01-任务编排规范.md`）：

| 能力 | 方案 |
| --- | --- |
| 定时 | Cron（支持年）+ 日历（节假日） |
| 依赖 | 上游任务成功 / 跨工作流依赖 / 数据到位感知 |
| 补数 | 按日期范围批量重跑，支持并行度 |
| 重跑 | 失败重试（默认 3 次）、手动重跑 |
| 告警 | 失败 / 超时 / 积压 → 消息 + 邮件 + 短信 + 钉钉 |
| 并发控制 | 工作流并行度、任务组并发、资源组配额 |
| 实时任务 | Flink 作业注册到 DS 监控视图 |
| 任务分组 | 按主题域、按优先级 |

## 9. 数据血缘

### 9.1 血缘级别

1. **表级**：源表 → 目标表；
2. **字段级**：源字段 → 目标字段（含转换表达式）；
3. **任务级**：由哪个任务产生；
4. **实体级**：关联指标、标签、主数据。

### 9.2 血缘抽取

- SQL 任务：使用开源解析器（Apache Calcite + 自研适配 PG/CK 方言） → AST → 字段级依赖；
- ETL DAG：直接读取 `pipeline.json`；
- Flink / Spark：作业提交时解析；
- 采集任务：源 → ODS 自动建立；
- 推送任务：ADS → 外部；
- 指标：从指标定义 SQL 解析；
- 外部（对接方）通过 `POST /lineage/ingest` 接口主动登记。

### 9.3 血缘存储

- PG `bdc_govern.t_lineage_node` / `t_lineage_edge`；
- 图查询：PG 递归 CTE（一期）；
- 二期：Nebula Graph，支撑大规模图查询。

### 9.4 血缘展示

- 画布：AntV G6；
- 下钻：表 → 字段；
- 上下溯：上游 3 跳 / 下游 3 跳；
- 影响分析：选中一字段，展示所有下游表 / 指标 / API；
- 变更影响：修改 DDL 时自动评估下游。

## 10. 数据地图

- 基于资产目录 + 血缘 + 质量 构建地图；
- 支持按业务域/部门/负责人/分级浏览；
- 全局搜索 + 推荐。

## 11. 典型流程：一个主题域的完整治理

```
1. 定义主题域 → 采集源库登记
2. 建 ODS 表 + 采集任务（含脱敏）
3. DWD 事实表建模 + SQL ETL 任务
4. DIM 维度从 MDM 同步 + SCD2
5. DWS 宽表 / ADS 场景表 + CK 物化视图
6. DQ 规则挂到每层关键表
7. 血缘自动建立 + 人工核对补齐
8. 指标登记口径 + 映射到 ADS
9. 资产目录上架 + 标签
10. 集市发布 / 服务 API 发布 / 大屏使用
```
