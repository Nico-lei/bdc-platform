# 01 数据采集模块详设

## 1. 模块职责

- 为数据中台提供**多源异构、实时 + 批量、全量 + 增量 + 拉链**的统一采集能力；
- 所有采集任务**界面化配置**、**元数据化存储**、**可视化监控**；
- 对接：关系型 / NoSQL / 文件 / API / MQ 五大类源。

## 4. 关键设计

### 4.1 连接器（Connector）插件化

采用 SPI 设计，每种数据源实现统一接口：

```java
public interface DataSourceConnector {
    String type();                 // mysql / pg / oracle / dm / ks / mongo / neo4j / es / redis / rest / soap / kafka / rocketmq ...
    ConnectResult testConnect(ConnectionConfig cfg);
    List<CatalogItem> listCatalogs(ConnectionConfig cfg);
    List<TableItem> listTables(ConnectionConfig cfg, String catalog);
    List<ColumnItem> listColumns(ConnectionConfig cfg, String catalog, String table);
    CollectJobSpec buildJobSpec(CollectJobRequest req);
}
```

每类连接器独立 jar 包，启动时通过 `ServiceLoader` 发现。国产数据库驱动（达梦 `Dm7JdbcDriver18`、金仓 `com.kingbase8.Driver`）单独打包在 `bdc-collect/libs/`，非开源需学校自行采购提供。

### 4.2 采集任务配置（元数据化）

任务主表 `bdc_collect.t_job`，关键字段（完整 DDL 见 `04-数据库详细设计`）：

| 字段 | 说明 |
| --- | --- |
| `job_code` | 任务唯一编码 |
| `job_name` | 中文名 |
| `src_conn_id` | 源连接 ID |
| `src_catalog` | 源库/源 Schema |
| `src_table` | 源表 |
| `tgt_schema` | 目标 schema（如 `ods_jwxt`） |
| `tgt_table` | 目标表 |
| `sync_mode` | FULL / INCR / CDC / LINK（拉链）|
| `incr_col` | 增量列（如 `update_time`） |
| `pk_cols` | 主键列（JSON 数组） |
| `cdc_startup` | `initial` / `earliest` / `latest` / `timestamp` |
| `filter_expr` | 行过滤表达式（SQL where） |
| `col_mapping` | 列映射 JSON（源列 → 目标列 + 转换表达式） |
| `desensitize_cols` | 脱敏列（参考安全模块规则） |
| `batch_size` | 单批条数 |
| `speed_limit` | 限速（KB/s） |
| `error_limit` | 错误容忍（条数/比例） |
| `schedule_type` | ONESHOT / CRON / REALTIME |
| `cron_expr` | Cron 表达式 |
| `depends_on` | 依赖任务 |
| `alert_cfg` | 告警配置 |
| `status` | 草稿 / 已发布 / 已停用 |

### 4.3 精细化控制

- **字段级**：列映射 + 列过滤 + 列转换（支持 SQL 表达式、内置函数：`SM4_ENC`、`SM3_HASH`、`TRIM`、`DATE_FMT`、`MASK_MOBILE`…）；
- **行级**：`filter_expr` where 条件 + 正则匹配；
- **分区裁剪**：源库超大表按日期/ID 范围裁剪；
- **限速**：并发数 + KB/s 限速，避免打挂源库；
- **重试**：阶段性重试（连接/查询/写入），最大 3 次，指数退避；
- **断点续传**：CDC 基于 Kafka 偏移；批量基于 `ckp_col` 游标；
- **幂等**：目标表主键 + `ON CONFLICT DO UPDATE`；
- **数据质量预检**：采集入 ODS 前执行"空值率 / 行数 / 主键唯一"三项基础检查。

### 4.4 全量 / 增量 / 拉链策略详解

#### 全量（FULL）
- 清空目标表（或按 `dt` 分区清理）再写入；
- 支持抽样（`SAMPLE 10000`）；
- 首次建立 ODS 推荐。

#### 增量（INCR）
- 基于游标列 `incr_col`，上次采集最大值 `ckp` 作为起点；
- 目标表按 `pk_cols` 合并（upsert）；
- 删除处理：源表若无"删除"语义（比如只更新状态列），通过 CDC 或定期全量对账补充。

#### 拉链（Slowly Changing Dimension Type 2）
- 每次采集识别变更：
  - 新增 → 插入 + `effect_from = now()`、`effect_to = 9999-12-31`、`is_current = true`；
  - 修改 → 旧记录 `effect_to = now()`、`is_current = false`；新记录同新增；
  - 删除 → 旧记录 `effect_to = now()`、`is_current = false`、`is_deleted = true`；
- 目标表结构统一加 `effect_from`、`effect_to`、`is_current`、`is_deleted`、`_hash`（字段快照 SM3）；
- 识别字段变化以 `_hash` 比对，避免逐字段 diff。

#### CDC（实时）
- Flink CDC 作业，读源端 binlog/WAL/LogMiner；
- 源端前置条件：
  - MySQL：`log_bin=ON, binlog_format=ROW, binlog_row_image=FULL, server_id 配置`；
  - PG：`wal_level=logical, max_wal_senders, max_replication_slots`；
  - Oracle：开启归档 + 补充日志 + LogMiner 用户授权；
- 输出 Kafka Topic：`bdc.ods.<src_system>.<table>`；
- 落地作业：Flink SQL `UPSERT INTO pg_sink`；
- 国产库（达梦/金仓）无稳定 CDC 接口，采用"时间戳轮询 + 触发器外置队列"的折中方案（由 collect 服务起 Python/Java 轮询任务）。

### 4.5 文件采集

- **发现**：本地目录 / FTP / SFTP / SMB，轮询 + inotify；
- **元数据提取**：文件名、大小、Hash SM3、MIME、业务字段（从文件名正则或 Sidecar JSON）；
- **内容提取**（可选，按文件类型）：
  - PDF / Office：Apache Tika 提取正文（离线需打包 tika + tesseract 中文）；
  - 图片 OCR：PaddleOCR（仅在需要时）；
- **存储**：原文件上 MinIO；元数据入 `bdc_warehouse.t_file_asset`；
- **命名规则**：`<bucket>/<domain>/<yyyy>/<mm>/<dd>/<uuid>.<ext>`；
- **去重**：按 SM3 Hash 跳过已入库文件。

### 4.6 API 采集

- 支持 REST（GET/POST/JSON/XML）、SOAP 1.1/1.2、WebService（WSDL）；
- 认证：Basic / Bearer / AK+SK / OAuth2 / 自定义 Header / 国密签名；
- 分页：游标 / Offset / Page；
- 增量：增量参数、时间戳；
- 响应 JSONPath / XPath 解析入目标表；
- 频率控制：QPS 限速；
- 失败重试：最大 3 次 + 失败队列（Kafka `bdc.collect.api.fail`），供人工介入。

### 4.7 MQ 采集

- Kafka / RocketMQ / RabbitMQ 消费；
- 消费组可复用业务既有；
- 反序列化：JSON / Avro / Protobuf（Schema Registry 可选）；
- 入 ODS 或直接进 DWD（由用户决定）；
- Offset 持久化 + 幂等写。

## 5. 采集任务生命周期

```
[草稿] ─发布─▶ [已发布] ─停用─▶ [已停用]
             └─启动─▶ [运行中] ─失败─▶ [失败] ─重跑─▶ [运行中]
                                  └─完成─▶ [完成]
```

- 草稿：可编辑、不可运行；
- 已发布：不可运行（等待调度）；
- 运行中：有活跃执行；
- 失败：有失败记录，可重跑；
- 完成：一次性任务运行完毕。

## 6. 调度对接

- 批量任务：下发到 DolphinScheduler；
- 实时任务：Flink 作业由 collect 服务直接提交（REST Rest Flink Gateway 或 Application Mode）；
- 任务状态由 DS 回调 `bdc-scheduler-adapter`；
- DS 日志 + Flink 日志接入日志中心 Loki，按任务聚合展示。

管理控制台"数据采集"模块提供：

| 功能 | 描述 |

## 8. 典型性能预算

| 场景 | 目标 | 配置建议 |
| --- | --- | --- |
| CDC 实时（单任务） | 延迟 ≤ 5s，吞吐 2~5 万 TPS | Flink Parallelism 2~4、Kafka 分区 6 |
| 全量同步（千万级表） | 30 min 内完成 | DataX channel 10，batchsize 10000 |
| 文件采集（10万文件） | 2h 内元数据化 | 多线程 + MinIO 并发上传 8 |
| API 采集 | 100 QPS/任务 | 并发 8、限速 QPS 50 |

## 9. 合规与安全

- 源端账号：最小权限（只读 + 明确 schema/表）；
- 采集链路：网络隔离 VLAN 40，源库 → 中台单向；
- 传输：强制 TLS（国密 SSL 优先），原源若不支持则 SSL/TLS；
- 敏感列：采集入 ODS 即脱敏或密文存储；明文进入 DWD 必须有审批；
- 审计：任何配置变更 + 执行记录入审计；
- 错误数据：不直接丢弃，写入 `bdc_collect.t_err_record`，供人工介入；
- 源端压力保护：限速 + 错峰 + 采集窗口（教务晚间、财务凌晨）。
