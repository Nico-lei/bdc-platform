# 04 数据推送模块详设

## 1. 模块职责

将数据中台（ODS/DWD/DWS/ADS 或中间结果）**安全、可控、可追溯地**推送到外部系统。

## 2. 支持的目标类型

| 类型 | 具体 | 全量 | 增量 | 实时 |
| --- | --- | --- | --- | --- |
| 数据库 | MySQL / PG / Oracle / 达梦 / 金仓 / SQL Server / ClickHouse / MongoDB | ✅ | ✅ | ✅（Flink） |
| API | REST (POST JSON / XML) | ✅ | ✅ | ✅ |
| MQ | Kafka / RocketMQ / RabbitMQ | — | — | ✅ |
| 文件 | SFTP / FTP / SMB / HTTP | ✅ | ✅ | — |

## 3. 架构设计

```
     治理层 / ADS 层 (PG / CK / MinIO)
              │
              │ 读取 (SQL/SELECT)
              ▼
     bdc-push 推送服务
         ├── 推送任务配置
         ├── 目标连接
         ├── 转换/脱敏规则
         └── 鉴权 & 限速
              │
              │ 提交执行
              ▼
     执行层 (DataX 批量 / Flink 实时 / HTTP Client)
              │
              ▼
        外部系统（数据库 / API / MQ / 文件）
              │
              ▼
        推送审计 + 回执 → bdc-audit
```

## 4. 推送任务模型

```
推送任务 PushJob
├── job_code / job_name
├── src (查询 SQL、ADS 表、视图、物化视图)
├── tgt_type (db / api / mq / file)
├── tgt_conn_id
├── tgt_table / tgt_endpoint / tgt_topic / tgt_path
├── push_mode (FULL / INCR / UPSERT / APPEND / REPLACE)
├── incr_col (若增量)
├── field_mapping (JSON)
├── transform_expr (字段级表达式，含 SM4_ENC、MASK_*)
├── filter_expr (where 条件)
├── schedule (CRON / 实时)
├── batch_size / qps / rate_limit
├── retry (max_retry, backoff, dead_letter_topic)
├── encrypt (字段级加密配置)
├── sign (请求签名: 国密 SM2 + SM3 / HMAC-SHA256)
├── audit_level (full / summary)
└── status
```

## 5. 推送模式

### 5.1 数据库推送
- 执行引擎：**DataX**（批量）/ **Flink JDBC Sink**（实时）；
- UPSERT：基于主键 `ON CONFLICT DO UPDATE`（PG）/ `ON DUPLICATE KEY UPDATE`（MySQL）等；
- 目标若无主键，使用 `INSERT`，不去重；
- 目标表由任务创建者手动建好；推送前可选"预检 schema"；
- 事务：DataX 批事务；Flink Sink Exactly-Once（需目标支持 XA）。

### 5.2 API 推送
- HTTP Client 基于 OkHttp；
- 每批 N 条（可配）序列化为 JSON 数组 POST；
- 鉴权：Basic / Bearer / AK+SK / OAuth2 / 自定义 Header / 国密签名；
- 失败重试 3 次，仍失败发到 Dead Letter Topic，人工介入；
- 响应校验：状态码 + JSONPath 匹配；
- 限速：QPS + 并发；
- 支持**签名字段**：Timestamp + Nonce + AppKey + 请求体 SM3 摘要 → SM2 签名，放在 `X-Sign` 头。

### 5.3 MQ 推送
- Kafka/RocketMQ/RabbitMQ 生产者；
- 支持消息键（partition 路由）；
- 支持事务消息（Kafka Transactional）；
- Schema：JSON / Avro；
- 支持加密 Payload（整体 SM4 加密，头部放 IV）。

### 5.4 文件推送
- 生成 CSV / JSON / Parquet / Excel；
- 输出到 SFTP / FTP / SMB / HTTP；
- 大文件分卷 + Sidecar MD5/SM3；
- 支持"目录轮换"模式；
- 文件内置水印（CSV 追加注释行、Excel Header 水印）。

## 6. 精细化控制

- 列级脱敏/加密；
- 行级过滤（where + 标签过滤）；
- 速率限制：防止打挂目标；
- 分批窗口：仅在业务低峰推送；
- 开窗补推：节假日补推；
- 目标回执：支持以回调或 MQ 方式接收目标的处理结果，更新推送状态。

## 7. 审计

- 每次执行写入 `bdc_push.t_push_run` 主记录；
- 每批次写入 `bdc_push.t_push_batch`，含条数、耗时、字节数、目标返回码；
- 不记录明细原始数据（大），仅记**抽样 Hash + 计数**；
- 所有推送敏感/机密级数据的任务需审批（二级 / 三级审批）；
- 推送前将数据打包后整体 SM3 Hash → 存储作为"数据出站哈希链"，供事后追查。

## 8. 数据出站安全闸门

- 所有推送任务发布前必须：
  1. 通过数据标准 / 脱敏规则校验；
  2. 安全管理员审批；
  3. 目标系统登记在"可推送白名单"（`bdc_push.t_tgt_whitelist`）。
- 触发以下条件自动拦截：
  - 目标 IP/Host 不在白名单；
  - 敏感字段明文推送且未豁免；
  - 单次推送量超过阈值（默认 100 万行或 1 GB）；
  - 推送时间窗口之外（默认 01:00~05:00）。

## 9. 监控与告警

- 执行成功率 / 失败率 / 积压 / 目标响应耗时 / QPS；
- 指标上报 Prometheus；Grafana 大盘；
- 触发告警：失败率 > 5%、积压 > 1 万、耗时 > 2×基线。
