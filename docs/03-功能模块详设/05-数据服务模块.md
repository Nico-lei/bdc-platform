# 05 数据服务模块详设

## 1. 模块职责

将数据能力以**标准化 REST API** 的形式对外共享；提供**在线配置 → 一键发布 → 鉴权限流 → 全链路监控**的能力。

## 2. 核心特性

- SQL → API：通过 SQL + 参数模板生成 API，无需写代码；
- API 版本化 / 生命周期管理；
- 鉴权：AK/SK + Token + 国密签名 + IP 白名单；
- 限流：QPS / 并发 / 配额（按应用/用户/API）；
- 缓存：Redis 缓存（按请求哈希）；
- 监控：调用量、耗时、错误率、Top N；
- 文档：OpenAPI 3 自动生成；
- 测试：内置调试台。

## 3. 架构设计

```
外部/内部调用方 ──▶ bdc-gateway ──▶ bdc-service-api
                                     │
          ┌──────────────────────────┼──────────────────────────┐
          ▼                          ▼                          ▼
     SQL 执行               缓存 (Redis)               调用记录 (Kafka→ES)
      │
      ▼
  PG / CK / MinIO
```

## 4. API 模型

```
API Definition
├── code / name / path (/api/dataservice/v1/{code})
├── method (GET/POST)
├── version (v1/v2)
├── description / owner
├── 数据源 (PG/CK/视图/存储过程)
├── 参数列表 (name, type, required, default, validate)
├── SQL 模板 (含 :param 参数)
├── 结果映射 (字段别名、类型、脱敏)
├── 鉴权方式 (AK/SK、Token、Open、国密签名)
├── 限流 (QPS, 并发, 日配额)
├── 缓存 (ttl, key 策略)
├── 分级 (L1~L4)
├── 生命周期 (开发/测试/已上架/下架)
└── SLA (P95 目标)
```

## 5. SQL → API 实现细节

- 编辑器：Monaco + SQL 语法 + 参数高亮；
- 参数绑定：预编译 PreparedStatement（避免 SQL 注入）；
- 结果处理：基于 `ResultSetMetaData` 生成 OpenAPI Schema；
- 脱敏：字段级脱敏规则（来自 security 模块） → 返回前统一应用；
- 分页：`limit` / `offset`（可由系统自动加） 或 游标；
- 空结果：返回 `data: []` 不报错；
- 超时：默认 5s，可调；
- 数据行上限：单次默认 1 万，可配置。

## 6. 鉴权体系

### 6.1 三种鉴权形态

1. **AK/SK（应用鉴权）**：为接入方/外部系统颁发 AK+SK；请求头 `X-App-Key`，签名头 `X-Sign`：
   - `signStr = method + "\n" + path + "\n" + sortedQuery + "\n" + bodySM3 + "\n" + timestamp + "\n" + nonce`
   - `X-Sign = HMAC-SM3(SK, signStr)`（或 SM2 私钥签名，SK 作为 SM2 私钥）；
2. **Token（用户鉴权）**：Gateway 校验 CAS Ticket 换 Token，后端透传；
3. **Open（匿名）**：仅 L1 公开数据，仍做 IP 限流。

### 6.2 应用管理

- 应用注册 → AK/SK 生成 → 管理员下发（SK 仅展示一次）；
- 应用关联可访问 API + 配额；
- 应用禁用、密钥轮换；
- 审计：每次调用登记应用、用户、API、耗时、状态。

## 7. 限流与熔断

- **Sentinel** 配置：
  - 资源：每个 API 为一个资源；
  - 规则：QPS / 并发 / 系统自适应；
  - 响应：429 `{"code":"RATE_LIMITED","msg":"..."}`；
  - 热点参数限流：对高风险参数（如单应用查询特定用户信息）做热点限流；
- **配额**：按日 / 月配额，超限降级或拒绝；
- **熔断**：错误率 > 50% 持续 10s 熔断 30s；
- **降级**：返回缓存 / 默认值 / 直接 503 友好提示。

## 8. 缓存策略

- 缓存 Key = hash(api_code + params + user_context + tenant)；
- 读命中 → 返回并在 Response Header 加 `X-Cache: HIT`；
- TTL 默认 60s；配置化；
- 支持"主动失效"（DDL / 数据变更事件驱动）；
- 敏感 L3/L4 数据不缓存。

## 9. 调用审计

- 每次调用事件写 Kafka `dataservice.call`；
- 消费者写入 Elasticsearch + 冷归档；
- 字段：应用、用户、API、参数（脱敏）、耗时、状态、IP、UserAgent、Trace ID；
- 保留：热 90 天（ES），冷 2 年（MinIO Parquet）。

## 10. 监控大盘

- 每 API：调用量 / 成功率 / P50/P95/P99 / Top 参数；
- 全局：QPS 时序、错误时序、调用应用 TopN、API TopN；
- 告警：P95 > SLA 2x、5xx 比率 > 1%、配额临近。

## 11. 文档与调试

- 发布成功后自动生成 OpenAPI 3 YAML；
- 在线调试台：输入参数 → 发送请求 → 查看结果；
- 导出 Postman Collection；
- 支持 Mock 模式（上线前）。

## 12. API 生命周期

```
开发中 ─▶ 测试 ─▶ 待审批 ─▶ 已上架 ─▶ 已下架
           ▲                              │
           └──── 申请变更 / 版本升级 ──────┘
```

- 版本策略：非兼容变更升版本号（`v2`）；
- 下架策略：提前 30 天通知已绑定应用 + 延期 90 天软下架。

## 13. 性能预算

| 场景 | 目标 |
| --- | --- |
| 简单单表查询 | P95 ≤ 50ms |
| 关联 2 表 | P95 ≤ 100ms |
| 聚合查询（CK） | P95 ≤ 500ms |
| 并发 | 100 并发稳定 |
| 单 API 日配额默认 | 10 万次 |

## 14. 接口（管理面）

```
POST /api/dataservice/v1/api/def       # 新建 API
PUT  /api/dataservice/v1/api/def/{code}
POST /api/dataservice/v1/api/def/{code}/publish
POST /api/dataservice/v1/api/def/{code}/offline
GET  /api/dataservice/v1/api/def?keyword=...
POST /api/dataservice/v1/app           # 注册应用
POST /api/dataservice/v1/app/{id}/key/rotate
GET  /api/dataservice/v1/metrics
POST /api/dataservice/v1/sandbox/call   # 在线调试
```

## 15. 实际调用面 URL 规则

```
GET/POST  /api/dataservice/{version}/{api_code}?<params>

示例:
GET /api/dataservice/v1/stu_info?student_no=20260101
Headers:
  X-App-Key: xxx
  X-Timestamp: 1714000000
  X-Nonce: abc123
  X-Sign: <SM3-HMAC 或 SM2 签名>
```
