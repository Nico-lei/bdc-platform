# 01 编码规范（Java / Vue / SQL）

## 一、Java 编码规范

遵循《阿里巴巴 Java 开发手册》（黄山版 2024）+ 以下补充条款。

### 1. 基础
- JDK 17；使用 `record`、`sealed`、`Text Block`、`var`（局部合理使用）；
- 文件头：
```java
/*
 * Copyright (c) 2026 <学校名称> 数据中台
 * SPDX-License-Identifier: Apache-2.0
 */
```
- 编码：UTF-8；换行：LF；缩进：4 空格；不使用 Tab。

### 2. 包命名
`cn.edu.<school>.bdc.<service>.<layer>`，`layer ∈ {controller, facade, application, domain, infrastructure, adapter, config}`。

### 3. 分层（DDD Lite）
```
controller   → HTTP 接入
application  → 用例编排、事务边界
domain       → 实体、值对象、领域服务、聚合根
infrastructure → 仓储实现、外部客户端、消息、数据库
adapter      → Feign / Kafka / 适配
config       → 配置与自动装配
```

### 4. 命名
- 类：大驼峰；接口无 `I` 前缀；
- 方法：小驼峰、动词开头；
- 变量：小驼峰、语义化，禁止单字母（循环 i/j 除外）；
- 常量：UPPER_SNAKE_CASE；
- 包名：全小写；
- DTO：`xxxDTO`；VO：`xxxVO`；Entity：`xxxPO`；Query：`xxxQuery`；Param：`xxxReq`；Return：`xxxResp`。

### 5. 异常
- 必使用业务异常：`BizException(code, msg)`；
- 全局异常处理 `@RestControllerAdvice`；
- 禁用 `throws Exception`、`catch Throwable`；
- 不吞异常；捕获后要么处理要么抛出。

### 6. 日志
- 统一使用 SLF4J；
- 日志级别：ERROR 告警、WARN 值得注意、INFO 业务关键路径、DEBUG 详细；
- 禁止输出敏感信息（身份证、手机、密码、密钥、SQL 参数原值）；
- 必带 `traceId`（由 Filter 注入 MDC）；
- 参数化：`log.info("stu={} created", stuId);` 禁止字符串拼接。

### 7. 事务
- `@Transactional(rollbackFor = Exception.class)`；
- 方法粒度；事务内不做远程调用、不做大文件处理；
- 超时：默认 5s；
- 分布式事务仅在必要时引入 Seata AT。

### 8. MyBatis / MyBatis-Plus
- XML 与 Mapper 分离；XML 放 `resources/mapper/`；
- 禁止 `${}` 拼接；用 `#{}`；
- 复杂查询放独立 XML；
- 通用 CRUD 用 `BaseMapper`；
- 关联查询返回 `DTO`，不暴露 PO。

### 9. Controller
- 一个方法一个 URL；
- 入参用 `@Valid` + Bean Validation；
- 返回统一 `R<T>`；
- 不在 Controller 写业务。

### 10. 并发
- `CompletableFuture` + 统一线程池 `bdcAsyncExecutor`；
- 禁止无界队列；
- 使用 `ReentrantLock`、`StampedLock`、`ConcurrentHashMap` 等；
- 禁用 `Executors.newFixedThreadPool` 无界队列。

### 11. 空值与 Optional
- 能返回空集合不返回 `null`；
- 参数校验：`Objects.requireNonNull`；
- `Optional` 只用于返回值。

### 12. 注解规范
- 自定义注解放 `bdc-common-core.annotation`；
- `@Audit` `@Desensitize` `@EncryptField` `@RateLimit` `@Idempotent` 等均有统一实现。

### 13. 代码审查要点
- Code Review 至少 1 人 +1；高危模块（安全/权限/加密） 2 人；
- 禁止 `TODO` 不带 Owner + 日期；
- 方法行数 ≤ 80；类 ≤ 500；复杂度圈复杂度 ≤ 10；
- Sonar：严重以上 0 容忍，Blocker/Critical 必修。

## 二、Vue 前端规范

### 1. 基础
- Vue 3.4 + `<script setup lang="ts">`；
- 目录：`src/{api,assets,components,composables,directives,layouts,router,stores,styles,types,utils,views}`；
- 命名：组件 PascalCase；文件 kebab-case（单文件组件用 PascalCase）；
- 组件 `<template>`、`<script setup>`、`<style scoped>` 顺序。

### 2. 状态管理
- Pinia 2；按领域 Store（如 `useAuthStore`、`useMetaStore`）；
- 避免在组件内直接调用 API，统一走 `api/` 层。

### 3. 类型
- 全项目 TS；禁用 `any`（除已知三方库）；
- API 响应结构由后端 OpenAPI 自动生成 `types/api.d.ts`。

### 4. 样式
- 主样式 SCSS + CSS 变量；
- Element Plus 主题定制在 `styles/element-vars.scss`；
- 深浅主题（对外暂默认浅色）。

### 5. 路由
- `vue-router` 4；懒加载；
- 权限路由：登录后按菜单动态挂载；
- 404 / 403 / 500 页预置。

### 6. Lint
- ESLint + Prettier + Stylelint；
- `husky + lint-staged` 预提交；
- Commit：Conventional Commits（`feat/fix/docs/style/refactor/test/chore`）。

## 三、SQL 规范

1. 关键字大写，对象名小写；
2. 一句一行或适当换行；
3. SELECT 不使用 `*`，显式列出；
4. JOIN 条件置 `ON`，过滤条件置 `WHERE`；
5. 注释：每段 SQL 顶部写清目的、输入、输出、责任人；
6. 禁止 `SELECT INTO ... FROM dual` 这类歧义语法；
7. CK 写入批量化；
8. 带分区的 SQL 必须带分区键过滤；
9. `DELETE`/`UPDATE` 必带 `WHERE`；禁止全表更新；
10. 大查询使用 `EXPLAIN ANALYZE` 评估；
11. ETL 任务 SQL 幂等：`INSERT ... ON CONFLICT DO UPDATE` / `INSERT ... SELECT ... WHERE NOT EXISTS`。

## 四、Git & 提交规范

- 分支：`main`（生产） `dev`（日常主） `feat/<jira-id>-xxx`（功能）`fix/<jira-id>-xxx`（修复）`release/<ver>`（上线）`hotfix/<ver>` ；
- PR 必须关联 Jira/Issue ID；
- PR 合并策略：squash（主干保持线性）；
- 提交信息：Conventional Commits；
- 禁止提交到 `main` 分支；
- 禁止提交秘钥、大文件（> 1MB 代码）、IDE 配置。

## 五、依赖规范

- 所有依赖来自学校内网 Nexus / Verdaccio；
- 使用 BOM 锁版本；
- 升级依赖需评估 CVE 与许可证；
- 禁用 GPL/AGPL/SSPL；
- 关键组件：Spring Boot、Alibaba SCA、PostgreSQL JDBC、Clickhouse JDBC、Flowable、Bouncy Castle 版本固化，升级走架构评审。

## 六、测试规范

- 单元测试 JUnit 5 + Mockito + AssertJ；核心模块覆盖 ≥ 70%；
- 集成测试 Testcontainers（PG、Kafka、MinIO、Redis）；
- 接口契约测试（RestAssured + JSON Schema）；
- 性能测试 JMeter / k6；
- 前端单测 Vitest；E2E Playwright；
- CI 必跑单元测试 + Sonar + 依赖扫描。
