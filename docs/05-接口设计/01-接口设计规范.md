# 01 接口设计规范

## 1. 总则

- **协议**：HTTP 1.1 / 2，传输层国密 TLS（对外）或 TLS 1.3（内网）；
- **风格**：REST + JSON，兼容部分 Streaming / WebSocket（血缘订阅、实时告警）；
- **格式**：请求/响应统一 UTF-8 JSON；错误码统一；字段统一 `camelCase`（前端友好）；
- **版本**：URL 路径版本（`/api/xxx/v1/...`）；重大不兼容变更升主版本；
- **命名**：资源名复数、动词用小写、路径使用 kebab-case。

## 2. URL 规范

```
https://api.bdc.edu.cn/api/{serviceDomain}/{version}/{resource}[/{id}][/{action}]

示例：
GET  /api/uac/v1/users/123
POST /api/collect/v1/jobs
POST /api/collect/v1/jobs/{code}/run
```

| serviceDomain | 业务领域（对外不暴露内部微服务名，做一层归并） |
| --- | --- |
| `auth`, `uac`, `metadata`, `mdm`, `indicator`, `tag`, `collect`, `warehouse`, `govern`, `push`, `dataservice`, `market`, `approval`, `visual`, `security`, `audit`, `notify`, `portal`, `file` |

## 3. 请求规范

### 3.1 Headers

| Header | 必需 | 说明 |
| --- | --- | --- |
| `Authorization` | 视场景 | `Bearer <JWT>` 或 `App AK=...` |
| `X-App-Key` | AK/SK 调用 | 应用 Key |
| `X-Timestamp` | AK/SK 调用 | Unix 秒时间戳，偏差 ±300s |
| `X-Nonce` | AK/SK 调用 | 随机串，服务端缓存 10min 防重放 |
| `X-Sign` | AK/SK 调用 | HMAC-SM3 或 SM2 签名 |
| `X-Trace-Id` | 建议 | 链路追踪 ID，不传则生成 |
| `Content-Type` | POST/PUT | `application/json;charset=UTF-8` |
| `Accept-Language` | 可选 | `zh-CN` 默认 |

### 3.2 分页

```
GET /api/xxx/v1/yyy?pageNo=1&pageSize=20&sort=createTime,desc
```

- `pageNo` 从 1 开始；
- `pageSize` 默认 20，最大 200；
- `sort=field,order`，多字段 `sort=a,desc&sort=b,asc`。

### 3.3 过滤 / 查询

采用 **URL 参数**（简单查询）或 **POST /search**（复杂查询）：

```
POST /api/metadata/v1/assets/search
{
  "keyword": "学生",
  "filters": {"layer":["DWD","DWS"], "subjectDomain":"SD_STU"},
  "pageNo": 1, "pageSize": 20,
  "sort": [{"field":"updateTime","order":"desc"}]
}
```

### 3.4 批量

```
POST /api/xxx/v1/yyy/batch      # 批量创建
PUT  /api/xxx/v1/yyy/batch      # 批量更新
POST /api/xxx/v1/yyy/delete     # 批量删除（用 POST，避免 DELETE 带 body）
```

## 4. 响应规范

### 4.1 统一结构

```json
{
  "code": "0",
  "msg": "success",
  "data": {},
  "traceId": "abc-123",
  "timestamp": 1714000000123
}
```

- `code`：字符串，`"0"` 成功；失败按下表；
- `msg`：中文提示；
- `data`：业务数据（对象 / 列表 / 分页对象）；
- `traceId`：链路追踪；
- `timestamp`：服务器毫秒时间戳。

### 4.2 分页响应

```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "list": [...],
    "pageNo": 1,
    "pageSize": 20,
    "total": 12345,
    "totalPage": 618
  }
}
```

### 4.3 错误码规范

三段式 `E-{Domain}-{Seq}`，HTTP Status 按语义返回（200 / 400 / 401 / 403 / 404 / 409 / 429 / 500 / 503）。

| 状态码 | code 前缀 | 示例 |
| --- | --- | --- |
| 200 | 0 | 成功 |
| 400 | E-COMMON-400xx | 参数错误、JSON 解析失败 |
| 401 | E-AUTH-401xx | 未登录、Token 失效 |
| 403 | E-AUTH-403xx | 无权限、数据权限拒绝 |
| 404 | E-COMMON-404xx | 资源不存在 |
| 409 | E-COMMON-409xx | 并发冲突、已存在 |
| 429 | E-SERVICE-429xx | 限流、配额用尽 |
| 500 | E-{Domain}-500xx | 服务内部错误 |
| 503 | E-COMMON-503xx | 熔断、维护中 |

示例：
```json
{
  "code": "E-COLLECT-40011",
  "msg": "采集任务编码已存在",
  "data": null,
  "traceId": "abc"
}
```

### 4.4 国际化

- 通过 `Accept-Language` 返回对应语言；
- 后端使用 `MessageSource`，`messages_zh_CN.properties` 必备，`messages_en.properties` 选备。

## 5. 安全

### 5.1 鉴权流程

- 用户面：CAS Ticket → JWT（签发中心化）→ 每次请求携带；
- 应用面：AK/SK → 签名 → 校验；
- 内部服务：mTLS + Feign 拦截器透传 Token / 服务身份。

### 5.2 签名算法（详）

```
stringToSign =
    HTTP_METHOD.toUpperCase() + "\n" +
    RAW_PATH + "\n" +
    sortedQueryString + "\n" +             # k=v&k=v（URL 编码，按 k 升序）
    bodyDigestHex + "\n" +                 # SM3(body) hex
    timestamp + "\n" +
    nonce + "\n" +
    appKey

# 方式一（推荐，性能好）
sign = HMAC_SM3(appSecret, stringToSign)

# 方式二（更强，审计友好）
sign = base64(SM2_Sign(appPrivateKey, stringToSign))
```

重放防护：服务端缓存最近 10 分钟的 (appKey, nonce) 集合。

### 5.3 参数防护

- SQL 注入：统一 PreparedStatement；禁用字符串拼接；
- XSS：响应头 `Content-Security-Policy`、HTML 转义；
- CSRF：纯 Token 场景天然免疫；Cookie 场景引入 `SameSite=Strict` + 自定义头校验；
- 文件上传：白名单扩展名、MIME 校验、服务端重命名、病毒扫描（ClamAV）；
- 下载越权：所有下载均经过权限校验与签发短链。

## 6. 日志与审计

- 所有接口**必须**打印入参（脱敏后）+ 耗时 + 状态；
- 高危接口（授权、下载、KMS、配置修改）**必须**写入审计事件；
- 链路追踪：TraceId 贯穿前端 → 网关 → 服务 → DB。

## 7. 速率限制

| 层级 | 粒度 |
| --- | --- |
| 全局 | 网关层 | 全站 QPS 默认 500 |
| 应用 | 按 AK | 默认 QPS 20 |
| 接口 | 按 api_code | 按配置 |
| 用户 | 按 userId | 敏感/高危接口 |
| 热点 | 按参数 | Sentinel 热点参数 |

超限响应：HTTP 429 + `Retry-After` + `X-RateLimit-Remaining`。

## 8. 兼容与弃用

- 弃用接口：Response Header `X-Deprecated: true; sunset=2026-12-31`；
- 弃用通知：门户 + 邮件 + 文档标红；
- 最低保留 **90 天**。

## 9. 契约先行（OpenAPI）

- 每服务随代码提交 `api/<service>.yaml`；
- CI 自动校验 `yaml` 正确性、字段一致性；
- 生成前端 TS 客户端 + Mock Server（用 `openapi-typescript` / `Prism`）；
- 对外接入方基于 OpenAPI 生成 SDK。

## 10. WebSocket / SSE

- 场景：实时告警、血缘订阅、大屏刷新；
- 路径：`wss://api.bdc.edu.cn/api/xxx/v1/ws/{topic}`；
- 鉴权：首帧 `{token: "..."}`；超时未鉴权断连；
- 心跳：30s；
- 消息体：JSON `{"type":"event","payload":{...}}`。

## 11. 模板返回对象（Java）

```java
public class R<T> {
    private String code;
    private String msg;
    private T data;
    private String traceId;
    private long timestamp;

    public static <T> R<T> ok(T data) { ... }
    public static <T> R<T> fail(String code, String msg) { ... }
}

public class PageVO<T> {
    private List<T> list;
    private int pageNo;
    private int pageSize;
    private long total;
    private int totalPage;
}
```

## 12. 大负载与流式

- 大批量导入：`multipart/form-data`，服务端边读边存；
- 大批量导出：SSE 或分页 + 去抖；导出结果入 MinIO，返回短链；
- 长耗时任务：202 Accepted + 任务 ID，前端轮询或订阅 SSE。

## 13. 幂等与重试

- `POST` 创建类：Header `X-Idempotency-Key`；服务端缓存 24 小时；
- 下游调用重试：指数退避（base=100ms, factor=2, max=3）+ Jitter；
- 幂等键命名：业务前缀 + 业务键。

## 14. 契约测试

- 基于 OpenAPI 的 Schema 校验；
- 后端单测使用 `RestAssured` + `json-schema-validator`；
- 前端 Mock：`Prism`；
- 端到端：Playwright（重点覆盖主流程）。
