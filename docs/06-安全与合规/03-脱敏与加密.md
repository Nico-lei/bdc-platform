# 03 脱敏与加密设计

## 1. 密码学总则

- 算法：**国密优先**（SM2 / SM3 / SM4）；必要兼容 AES-256-GCM / SHA-256 / RSA-2048；
- 密钥长度：SM2 256 bit、SM4 128 bit、SM3 256 bit；
- 模式：SM4 GCM（推荐）、CBC（兼容）；严禁 ECB；
- 随机数：SecureRandom + 国密 RNG；
- 填充：PKCS7；
- IV：12~16 字节随机，绝不复用；
- 签名：SM2 WITH SM3；
- 哈希：SM3（64 位 Hex）；敏感哈希加盐（`PBKDF2-SM3`，迭代 ≥ 10 万次）。

## 2. 密钥分层管理

```
          Master Key (MK)
          存 HSM（或软 HSM）
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
  DEK_Biz    DEK_Audit   DEK_Backup    (按业务域派生)
    │
    ├─ FEK_Student_IdCard  (字段密钥)
    ├─ FEK_Student_Mobile
    ├─ FEK_Staff_Salary
    ...
```

- **MK**：年度轮换，HSM 保管；密钥材料不出卡；
- **DEK**：半年轮换；存库（`bdc_security.t_kms_key`）MK 加密；
- **FEK**：按需派生；存库同上；
- 轮换保留旧版本 **3 年**，旧数据继续可解密；
- 密钥操作全部审计（双人复核 + MFA）。

## 3. 数据分类分级细则

### 3.1 定级准则

| 维度 | 说明 | 示例 |
| --- | --- | --- |
| 敏感性 | 一旦泄露造成的损害 | 公开/内部/严重/重大 |
| 影响对象 | 个人/组织 | 个人信息+家庭地址 → 个人 |
| 法律合规 | 是否受法规管控 | 个人信息、涉密材料 |
| 业务重要性 | 是否核心业务 | 身份证号、成绩 |

### 3.2 分级表（节选）

| 字段类别 | 级别 | 处置 |
| --- | --- | --- |
| 用户 ID、姓名 | L3 敏感 | 脱敏 |
| 身份证号 | L4 机密 | 加密 + 脱敏 + 申请审批 |
| 手机号、邮箱 | L3 | 加密+脱敏 |
| 家庭住址 | L3 | 脱敏 |
| 工资、绩效 | L4 | 加密 + 双审 |
| 成绩、考评 | L3~L4 | 脱敏，L4 加密 |
| 简历、档案 | L4 | 加密 + 文件级水印 |
| 学号 / 工号 | L2 | 可明文 |
| 部门名称 / 专业名 | L1 | 公开 |
| 统计汇总 | L1~L2 | 公开 |
| 密码、密钥、Token | L4 | Hash/加密 + 绝不返回 |

### 3.3 自动识别规则库（节选）

| 名称 | 正则 / 检测 | 默认级 |
| --- | --- | --- |
| 中国手机号 | `1[3-9]\d{9}` + 长度=11 | L3 |
| 身份证号 | 15/18 位 + 校验位 | L4 |
| 邮箱 | RFC 邮箱正则 | L3 |
| 银行卡 | Luhn 校验 + 开头 BIN 库 | L4 |
| 车牌 | 中文 + 字母+数字 7 位 | L3 |
| 姓名 | 中文 2-4 字 + 字典匹配 | L3 |
| 工资关键字 | 列名/注释包含 salary/wage/pay | L4 |
| 病历关键字 | 列名包含 health/medical/disease | L4 |
| 密码 / Token | 列名 password/token/secret | L4 |

## 4. 脱敏

### 4.1 算法细节

| 算法 | 规则 |
| --- | --- |
| MASK_NAME | 保首末字 + 中间 `*`；单字"*"；两字第二字"*" |
| MASK_MOBILE | 保前 3 后 4，中间 `****` |
| MASK_IDCARD | 保前 4 后 4，中间 `****`；18 位维持长度 |
| MASK_EMAIL | local 前 1 + `***` + `@` + domain |
| MASK_ADDR | 保前 6 字符 + `****` |
| MASK_BANK | 保前 4 后 4 |
| HASH_SM3 | SM3 Hex，可选加盐 |
| RANDOM | 同类型随机，不保留原始分布 |
| DATE_SHIFT | 同记录平移 ±N 天（N 随机，同用户一致） |
| RANGE_ROUND | 数值按精度四舍五入 |
| TRUNCATE | 按长度截断 |

### 4.2 脱敏规则优先级

1. 用户级 > 部门级 > 角色级 > 全局默认；
2. 申请通过的临时授权（`t_data_grant`）可覆盖默认规则；
3. 审计敏感字段一律脱敏；
4. 导出文件默认按最严格规则。

### 4.3 实现点

- **API 返回**：DTO 字段上 `@Desensitize(algo=MASK_MOBILE, levelCheck=true)` 注解 → AOP 统一处理；
- **SQL 查询**：MyBatis 插件在返回 ResultSet 时替换；
- **大查询**：结果集 Stream 处理，防 OOM；
- **文件导出**：Excel/CSV 写入时注入脱敏；
- **大屏/BI**：数据集 SQL 预聚合 + 视图层脱敏；
- **缓存**：脱敏后值不复用（含用户 ID / 角色信息）。

## 5. 加密（字段级 / 整体）

### 5.1 字段级

```java
@EncryptField(alg=SM4_GCM, keyId="FEK_STU_IDCARD")
private String idCard;            // 存储: base64(密文); 查询: hash 副列
private String idCardHash;        // SM3(idCard + 盐)，用于等值查询
```

- 密文列存储密文，明文列不存；
- 模糊查询不支持字段加密（若必须，需审批使用 FPE 格式保留加密，或降级为索引表）；
- 序列化：`{alg,keyId,ver,iv,ct,tag}` base64。

### 5.2 整体加密（文件 / 备份）

- 文件：上传时服务端 SSE（MinIO SSE-C / SSE-S3 + 国密 KMS 插件），密钥 DEK 按桶派生；
- 备份：pgBackRest / clickhouse-backup 启用加密，密码由 KMS 注入；
- 归档：写 MinIO 归档桶前压缩 + SM4 加密。

### 5.3 传输加密

- 对外：国密 TLCP（GM/T 0024-2014，基于 TLS 1.2 的国密改造）+ 兜底 TLS 1.3（RSA/ECC 套件）；
- 内网：mTLS，证书 PKI 自签；
- DB 连接：
  - PG：`sslmode=verify-full`；
  - CK：TLS；
  - Kafka：SSL + SASL；
  - MinIO：HTTPS；
- DataX / Flink CDC：启用源端 TLS；必要时使用 SSH 隧道。

## 6. 密钥管理服务（KMS）

### 6.1 API

```
POST /api/security/v1/kms/dek          {scope,algo}  → 返回 keyId
POST /api/security/v1/kms/dek/{id}/rotate
POST /api/security/v1/kms/envelope/encrypt  {keyId, plaintext (base64)}  → 密文
POST /api/security/v1/kms/envelope/decrypt  {keyId, ciphertext}
```

- 调用需：服务 mTLS + 服务账号 + 审计日志；
- 人类管理员操作需：MFA + 二人复核（高危）。

### 6.2 实现

- SDK：`bdc-common-security` 封装；
- 缓存：本地进程缓存 DEK 5 min（防止雪崩）；
- 本地缓存加锁保护，进程结束清零；
- 性能：加解密 1KB 数据 ≥ 5 万 QPS / 节点（国密加速卡可上到十万级）。

## 7. 水印实现

### 7.1 可见水印

- Web 页面：`position:fixed; pointer-events:none; transform:rotate(-25deg); opacity:0.08`；
- PDF：PDFBox 逐页加文本 + 阴影；
- Excel：Apache POI 页面页眉页脚 + 图层；
- CSV：导出前在文件首尾添加注释行（姓名/工号/时间/IP）；
- 图片：ImageMagick / Java Graphics2D 叠加；
- 视频：ffmpeg `drawtext` 滤镜实时覆盖（仅点播流）。

### 7.2 隐写水印

- 图片 LSB：修改像素 LSB 位编码"用户 ID+时间戳"；
- 频域（DCT）：更抗攻击；
- PDF/Word：元数据 + 不可见字符 + 零宽字段；
- CSV：添加少量非关键记录微扰（可配置关闭）；
- 视频：帧级元数据嵌入；
- 解码：`POST /api/security/v1/watermark/extract` 上传样本 → 返回溯源信息。

### 7.3 适用策略

| 文件/场景 | 可见 | 隐写 |
| --- | --- | --- |
| L1 公开资料 | 否 | 否 |
| L2 内部 | 是 | 否 |
| L3 敏感 | 是 | 是 |
| L4 机密 | 是（强水印） | 是（多算法混合） |
| 大屏/BI 页面 | 是 | — |
| 导出文件 | 是 + 隐写 | 是 |

## 8. 关键场景与流程

### 8.1 新建表后自动分级

```
1. 捕获 DDL 事件
2. 扫描列名 + 注释 + 样本
3. 命中规则 → 打分 → 建议级别
4. 写 t_col_class (confirmed=false)
5. 通知数据管理员复核
6. 复核通过 → confirmed=true + 广播事件
7. 各消费服务刷新本地缓存
```

### 8.2 API 返回脱敏

```
Client → Gateway → Service → DTO 装配
 → DTO 中 @Desensitize 生效
 → 根据当前 user + col_rule 合并 → 应用算法 → 返回
```

### 8.3 高危解密调用

```
业务请求 → KMS decrypt → 校验：服务身份 + IP + 限流 → 返回 → 审计
```

### 8.4 导出 CSV / Excel

```
1. 权限检查（表+字段）
2. 行列规则应用
3. 分页读取
4. 逐批写入 POI / CSV
5. 首页附生成时间、用户、文件 Hash
6. 嵌入水印（可见 + 隐写）
7. 上传 MinIO
8. 返回短链 + 审计
```
