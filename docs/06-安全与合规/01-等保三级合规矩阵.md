# 01 等保三级合规矩阵

> 依据：GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》第三级（S3A3G3）。
> 本矩阵列出**每一条技术与管理要求**的落实方案与责任模块，作为测评对接材料。

## 一、安全物理环境（由学校机房承担，中台协同）

| 条款 | 要求 | 落实 |
| --- | --- | --- |
| 8.1.1 物理位置的选择 | 机房选址 | 学校 IDC 机房 |
| 8.1.2~8.1.10 | 物理访问/防盗/防雷/防火/防水/防潮/防静电/温湿度/电力供应/电磁防护 | 学校 IDC 达标 |

## 二、安全通信网络

| 条款 | 要求 | 落实 | 模块 |
| --- | --- | --- | --- |
| 8.2.1.a | 保证网络设备可用性 | 核心交换冗余、HSRP/VRRP | 网络 |
| 8.2.2.a | 采用校验技术保证通信完整性 | 国密 TLS / TLS1.3，IPsec VPN | 网关/中间件 |
| 8.2.2.b | 采用密码技术保证保密性 | SM4 加密敏感数据；SM2 国密 TLS | bdc-security |
| 8.2.3 | 网络边界防护（DMZ、反代、WAF） | Nginx 反代 + WAF + VLAN 隔离 | 部署 |

## 三、安全区域边界

| 条款 | 要求 | 落实 | 模块 |
| --- | --- | --- | --- |
| 8.3.1 边界防护 | 防火墙默认拒绝 | 三层 VLAN + 默认拒绝 + 最小化白名单 | 部署 |
| 8.3.2 访问控制 | 按业务需要授权 | 硬件防火墙 + 应用层 RBAC | 部署/UAC |
| 8.3.3 入侵防范 | IDS/IPS | 部署 IDS（Suricata/Snort）+ 应用层 WAF 规则 | 部署 |
| 8.3.4 恶意代码防范 | 终端/服务器杀毒 | ClamAV + 定期扫描 + 文件上传扫描 | 部署/文件服务 |
| 8.3.5 安全审计 | 边界审计 | 防火墙日志集中，保留 ≥ 180 天 | 部署 |

## 四、安全计算环境

| 条款 | 要求 | 落实 | 模块 |
| --- | --- | --- | --- |
| 8.4.1 身份鉴别.a | 身份唯一，鉴别信息复杂度 | CAS + LDAP + 密码策略 | bdc-auth |
| 8.4.1.b | 登录失败处理、限制非法登录 | 5 次锁 30 min、IP 锁 | bdc-auth |
| 8.4.1.c | 网络远程管理两种及以上鉴别技术 | 管理员强制 MFA（TOTP）| bdc-auth |
| 8.4.2 访问控制.a | 账户按需分配、默认账户修改 | 三权分立、无 default；LDAP 同步 | bdc-uac |
| 8.4.2.b | 授予账户最小权限 | 角色 + 数据权限最小化 | bdc-uac |
| 8.4.2.c | 授权主体配置访问控制 | 部门/数据管理员审批 | bdc-market/approval |
| 8.4.2.d | 访问控制粒度到主体级、客体文件、数据库表级 | 库/表/行/列 四级 | bdc-uac |
| 8.4.2.e | 敏感信息访问需授权 | 分级 + 审批 | bdc-market/security |
| 8.4.3 安全审计.a | 覆盖重要事件 | 登录/授权/数据访问/下载/配置 | bdc-audit |
| 8.4.3.b | 审计记录包含足够信息 | who/when/where/what/result | bdc-audit |
| 8.4.3.c | 审计记录保护、防删改 | Hash 链 + 只追加表 + 定期签名 | bdc-audit |
| 8.4.3.d | 三权分立 | sys / sec / audit | bdc-uac |
| 8.4.4 入侵防范 | 最小化服务、关闭默认端口、及时补丁 | 镜像基线 + 漏洞扫描 | 部署 |
| 8.4.5 恶意代码防范 | 安装反病毒 | ClamAV + 文件扫描 | 部署/文件服务 |
| 8.4.6 可信验证 | 启动可信 | BIOS UEFI Secure Boot（硬件） | 学校/硬件 |
| 8.4.7 数据完整性.a | 鉴别信息/重要数据传输完整性 | SM3 摘要 + 签名 | bdc-security |
| 8.4.7.b | 存储完整性 | SM3 校验 + WAL/备份校验 | 存储 |
| 8.4.8 数据保密性.a | 传输加密 | 国密 TLS | 网关 |
| 8.4.8.b | 存储加密 | 字段级 SM4 + OS 层 LUKS | bdc-security |
| 8.4.9 数据备份恢复.a | 本地备份 + 独立介质备份 | 本地 + 校内独立介质冷备；具备条件时再扩展异地 | 运维 |
| 8.4.9.b | 热冗余 | 主备流复制、CK 多副本 | 运维 |
| 8.4.10 剩余信息保护 | 删除后不可恢复 | 逻辑删除 + 物理清理策略 | bdc-warehouse |
| 8.4.11 个人信息保护 | 仅采集必要、告知、授权 | 分类分级 + 申请审批 | bdc-security |

## 六、安全管理制度

由学校 PMO + 实施方共同编制，含：

- 安全总方针、安全策略；
- 制度分类清单（运维、开发、应急等）；
- 制定/发布/评审/修订流程；
- 落地：上墙、培训、考核。

## 七、安全管理机构

| 岗位 | 职责 |
| --- | --- |
| 安全领导小组 | 决策 |
| 安全管理员 | 日常执行 |
| 审计管理员 | 审计 |
| 系统管理员 | 运维 |
| 应急响应小组 | 事件处置 |
| 内审小组 | 合规审查 |

## 八、安全管理人员

- 岗位责任书 + 保密承诺；
- 入职背调（关键岗位）；
- 年度安全培训 + 考核；
- 离岗交接清单与审计。

## 九、安全建设管理

- 立项 / 定级 / 备案（公安网安）；
- 采购要求、开发过程安全（本文档 + 开发规范）；
- 测评整改；
- 服务供应商管理。
